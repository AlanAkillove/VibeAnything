# 05-AI 辅助文献调研

> 读完本文你将了解：如何从一篇好论文出发做「滚雪球」式搜索、如何用 AI 辅助总结和对比论文、以及如何让 AI 帮你找 related work 的脉络。

---

## 你大概率遇到过的问题

做文献调研时常见的困境：

- **开头不知道怎么搜**——Google Scholar 上搜关键词出来几千篇，不知道从哪下手
- **读了一大堆独立论文，但串不起来**，不知道这些工作的演进关系
- **相关工作的范围很难把握**——该读到多细？该覆盖多广？
- **对比相关论文很痛苦**，每篇看一遍然后手动列表格
- **写完 related work 回头看，发现漏了好几篇重要工作**

文献调研的核心不是「读所有论文」，而是 **从一篇高质量种子论文出发，沿着引用网络高效扩展**。

---

## 常见的误解

| 你以为的 | 实际的 |
|---------|--------|
| 文献调研就是从关键词搜索开始 | 更高效的策略是先找到 1-2 篇核心论文，再滚雪球扩展 |
| 论文读得越多，调研越全面 | 读 50 篇但没串起来，比读 15 篇但有清晰脉络差很多 |
| AI 不能帮你看论文，只能帮你翻译 | AI 可以用来做对比、找趋势、找 gap，精度取决于你怎么用 |
| Related work 要尽可能全 | Related work 的核心是定位你的贡献，不是炫耀列了多少篇 |

## 滚雪球搜索法

### 第一步：找到种子论文

种子论文的质量决定了后期扩展的质量。怎么找：

- **顶会 best paper + 高引工作**，这些基本是领域基石
- **Survey paper**，读一篇顶刊 survey 能覆盖几十篇重要工作
- **你的 advisor/同事推荐的 2-3 篇**
- 用 Google Scholar 读一篇 survey 时，把 survey 引用的论文当种子

### 第二步：前向搜索（Forward search）

找到种子论文后，看「谁引用了这篇论文」：

- Google Scholar 的 **Cited by** 按钮
- Semantic Scholar 的引用网络可视化

这一轮能找到这篇论文的后续改进工作。特别关注：

- 有显著引用量的后续工作（说明拿到了社区认可）
- 在相关工作 section 中被反复提及的工作（说明它们是关键节点）

### 第三步：后向搜索（Backward search）

看种子论文的 References 部分：

- 种子论文引用了哪些相关工作
- 引用多的工作通常是该领域的 key paper
- 种子论文自己做的 related work section 本身就是一份精筛过的列表

**经典模式**：一篇种子论文 -> 前向搜 10 篇引用者 -> 后向搜 20 篇参考文献 -> 这 30 篇再过一遍相关引用 -> 覆盖 50-100 篇核心论文

### 第四步：用 AI 加速滚雪球

```python
# 让 AI 总结论文并推荐扩展方向
prompt = f"""
我有一篇论文《{paper_title}》，下面是我的笔记（含问题和方法）：
{my_notes}

请帮我：
1. 简要总结这篇论文的核心贡献
2. 推荐与这篇论文最相关的 5-8 篇后续/改进工作
3. 推荐这篇论文的基础工作
4. 指出目前该领域可能还有哪些 open problem
"""
```

## AI 辅助论文对比

读了一堆论文后，需要横向对比。这是 AI 最擅长的：

### 对比表格生成

假设你读了 5 篇关于 LLM alignment 的论文，让 AI 帮你整理：

```markdown
## 论文对比

| 论文 | 方法 | 数据集 | 核心 metric | 优势 | 局限 |
|------|------|--------|-------------|------|------|
| Paper A | RLHF | ... | reward score | ... | ... |
| Paper B | DPO | ... | win rate | ... | ... |
| Paper C | KTO | ... | ... | ... | ... |
| Paper D | ... | ... | ... | ... | ... |
```
### 研究脉络梳理

让 AI 帮你梳理研究方向的时间线：

```
2017: Transformer 提出
  -> 2018: GPT / BERT（预训练时代开始）
    -> 2019: GPT-2 / RoBERTa / T5（更大更强）
      -> 2020: GPT-3 / Scaling Laws（规模涌现）
        -> 2021: InstructGPT / RLHF（对齐）
          -> 2022: ChatGPT / LLaMA（开源大模型）
            -> 2023: DPO / 4-bit 量化 / MoE ...
```

### 找研究 gap

AI 可以帮你分析已有工作中：

- **哪些问题还没解决**（如：当前方法在 low-resource languages 上效果不好）
- **哪些假设可能不成立**（如：scaling laws 在小数据 regime 下不适用）
- **哪些方法有理论缺陷**（如：RLHF 的 reward hacking 问题）

## AI 辅助搜索的具体工具

| 场景 | 工具 | 使用方式 |
|------|------|----------|
| 找论文 | Semantic Scholar API / Elicit | API 调用或网页端搜索相关论文 |
| 总结论文 | ChatPDF / Humata / NotebookLM | 上传 PDF 直接提问 |
| 对比论文 | ChatGPT / Claude | 喂多篇论文摘要，要求对比 |
| 写 related work draft | ChatGPT / Claude | 提供论文列表和笔记，生成初稿 |
| 找 citation | Zotero + AI 插件 | Zotero 里用 AI 插件分析论文库 |

## 什么时候该停下来

文献调研最容易踩的坑是 **永远觉得还不够**。

停下来的信号：

1. 新读的论文已经没有让你预料之外的发现
2. 你能画出一张清晰的研究脉络图（三四个流派/方法）
3. 你能说出当前领域的 open problem（至少 2-3 个）
4. 你能判断自己的方法和已有工作的差异

达到这四个信号，就可以停下来了。剩下的工作可以在写论文的过程中补。

---

## 要点总结

1. **文献调研从 1-2 篇高质量种子论文开始**，顶级 survey 是最佳入口
2. **滚雪球搜索**：前向（谁引用了这篇）+ 后向（这篇引用了谁），两轮覆盖核心文献
3. **AI 辅助三大用途**：总结单篇论文、生成对比表格、梳理研究脉络
4. **用 AI 找 gap**：分析已有工作的局限，找出尚未解决的问题
5. **不要无限调研下去**：能画出脉络图 + 说出 open problem + 判断自己方法差异时就可以停了
6. **最有效的方法是用 AI 加速整理和对比，把精力集中在判断和决策上**
