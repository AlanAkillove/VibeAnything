# 03-高效阅读与管理论文

> 读完本文你将了解：论文搜索渠道（Google Scholar、Semantic Scholar、ArXiv）、精读和泛读的区分方法、Zotero 和 Obsidian 的论文笔记体系搭建。

---

## 你大概率遇到过的问题

科研新手面对论文时常见的困境：

- **论文搜不到**：在知网/百度学术搜了半天，发现好论文都在 ArXiv 和顶会里
- **看到感兴趣的论文不敢确定质量**：分不清顶会 paper 和 workshop paper
- **打开一篇论文从头读到尾，读完发现没什么用**——花了两小时浪费了
- **读过的论文过两周就忘**，想引用时找不到笔记
- **论文越来越多**，桌面上一堆 PDF 不知道谁是谁
- **经典论文和最新论文之间怎么平衡**，该追新还是先补齐基础

这些问题的核心是：**你不会管理论文的输入流**。好的论文阅读方法不是「读更多」，而是「读更聪明的」+「读完能记住」。

---

## 常见的误解

| 你以为的 | 实际的 |
|---------|--------|
| 读论文越多越好 | 读 100 篇水论文不如精读 10 篇高引论文 |
| 论文应该从头读到尾 | 好的阅读方法是从后往前看、按需精读 |
| 所有论文都用同一方法读 | 论文分「泛读」「粗读」「精读」三档，方法不同 |
| ArXiv 上的论文都是学术严谨的 | ArXiv 是无审稿预印本，质量全靠你自己判断 |

## 论文搜索策略

### Google Scholar

最核心的论文搜索引擎。几个技巧：

- **Cited by**：引用数是最直观的质量信号（但别忘了，2024 年的论文和 2010 年的论文引用数不可比）
- **Related articles**：找到一篇好论文后，点这个能发现同领域的其他重要工作
- **Create alert**：关键词建 Alert，新论文自动推送
- 用 `"exact phrase"` 或 `author:"author name"` 精确搜索

### Semantic Scholar

比 Google Scholar 多了 AI 辅助功能：

- 直接总结论文的 TL;DR
- 可视化引用网络（谁引用了谁、哪些是最重要的引用）
- 自动提取核心贡献和方法

### ArXiv

论文最新发布地，每天更新几百篇。建议配合 RSS 或工具使用：

- **ArXiv Sanity**：Andrej Karpathy 做的，按你的阅读偏好推荐论文
- **ArXiv API / email alert**：按关键词每天接收新论文

### 判断论文质量的信号

| 信号 | 高 | 中 | 低 |
|------|-----|-----|-----|
| 出版地 | NeurIPS / ICML / ICLR / CVPR / ACL / AAAI | 其他 CCF 会议 / 顶会 Workshop | ArXiv only（无后续 publication） |
| 引用数（2018-2022） | 100+ | 20-100 | < 20 |
| Author reputation | 你经常在优秀论文里看到的名字 | 新 PI 或博士生 | 非学术机构 |
| Code released | 有，且维护良好 | 有但代码比较乱 | 没有 code |

## 三档阅读法

### 泛读（每天 10-20 篇）

看标题 + 摘要 + 结论，回答三个问题：

1. 这篇论文研究什么问题？
2. 方法核心思路是什么？
3. 结果比 baseline 好多少？

每篇 3-5 分钟，筛掉 90% 的论文。值得往下读的扔进 Zotero 收藏夹。

### 粗读（每周 5-10 篇）

看 Introduction + Method overview（通常是 Figure 2）+ Experiments table。关注：

- Problem formulation
- Method 的高层思路（不用看公式细节）
- Main results 和 baselines
- Ablation study 结果

每篇 15-20 分钟。值得精读的标记出来。

### 精读（每周 1-3 篇）

完整读全文，包括公式和实验细节。尤其关注：

- Related work 部分（帮你了解研究脉络）
- Method 的具体实现细节
- 实验设置的完整描述（数据集、超参数、评估指标）
- Limitations（论文自己的局限意识）
- 补充材料（Appendix）

每篇 60-90 分钟。精读要写笔记。

## 论文笔记体系：Zotero + Obsidian

### Zotero 做文献管理

Zotero 的核心功能：

- **一键抓取**：浏览器插件抓取论文元数据（标题、作者、期刊、摘要）
- **PDF 管理**：直接存 PDF，支持高亮和注释
- **Tag 和 Collection**：按主题分类，打标签（如 `#LLM`、`#attention`、`#method`）
- **自动生成引用**：拖拽就能复制 BibTeX 格式

```bash
# Zotero + Zotfile 插件自动重命名 PDF
格式：Author_Year_Title.pdf
例：Vaswani_2017_Attention Is All You Need.pdf
```

### Obsidian 做知识管理

读完论文后，在 Obsidian 里写文献笔记。推荐的结构：

```markdown
# Vaswani 2017 - Attention Is All You Need

**Tags**: #transformer #attention #NLP #arch

## 核心贡献
提出纯 attention 机制，完全替代 RNN/CNN 的 sequence transduction 架构

## Method 要点
- Scaled Dot-Product Attention: QK^T / sqrt(d_k)
- Multi-Head Attention: 对 Q、K、V 做 h 个投影并行算 attention
- Positional Encoding 代替序列位置信息

## 和已有工作的关系
- 之前的 seq2seq 用 RNN + attention（Bahdanau 2014, Luong 2015）
- Transformer 是第一篇彻底去掉 RNN 的

## 实验结果
- WMT 2014 EN-DE: BLEU 28.4（SOTA）
- 训练速度快于所有 RNN-based 模型
- 不同 head 学到不同 attention pattern

## 我的想法
- 这篇后续衍生出了 BERT、GPT、ViT 等大量工作
- 现在看很多设计选择（如 LayerNorm 的位置）已经有更好的替代了
```

### 建立论文图谱

在 Obsidian 里建立论文间的关联：

- 在笔记中 `[[引用其他论文]]` 自动建立链接
- Graph View 可视化学术脉络
- 按 timeline 梳理领域发展

这种做法的价值：**读过的论文不是孤立的知识点，而是可导航的网络**。

---

## 要点总结

1. **论文搜索三件套**：Google Scholar（最全）、Semantic Scholar（筛选）、ArXiv（最新）
2. **论文质量看 venue + citations + author + code release** 四个信号
3. **三档阅读法**：泛读（10%时间）-> 粗读（30%时间）-> 精读（60%时间），大多数人错在什么论文都精读
4. **Zotero 做 PDF 管理和引用生成**，Obsidian 做知识笔记和文献图谱
5. **精读的论文一定要写笔记**，结构包括核心贡献、Method 要点、和已有工作的关系、你的想法
6. **建立论文之间的关联**，让知识成为可导航的网络而不是零散文件
