# 02-隐私与数据安全

> 读完本文你将了解：你用 AI 时你的数据去了哪里、隐私风险有哪些、怎么保护你自己和用户的数据、以及作为开发者你应该注意什么

---

## 你可能遇到过的问题

你用 AI 做了很多事情，但你有没有想过：

- **你把代码贴给 AI 让它帮你调试**——这段代码会不会被保存下来？会不会被别人看到？
- **你让 AI 帮你写邮件**——邮件里的个人信息会被用在哪里？
- **你看到 AI 平台说会用你的数据来改进模型**——这是不是意味着你发的所有内容都会被用来训练？
- **你想做一个 AI 应用**——但用户的数据你怎么保护？出了事谁负责？

这些问题没有简单的"是/否"答案——但你需要知道**发生了什么**以及**你能做什么**。

---

## 常见的误解

| 你以为的 | 实际的 |
|---------|--------|
| 你用 AI 干啥平台都知道 | 平台能看到你发了什么（为了处理你的请求），但 API 调用默认不会被用来训练（除非你手动 opt in）|
| AI 平台会保护你的数据隐私 | 数据隐私的保护水平取决于平台的服务条款和你选择的设置 |
| 数据安全就是不要泄露密码 | 数据安全还包括：训练数据的归属、模型是否会记忆敏感信息、输出是否会暴露用户数据 |
| 中国有《个人信息保护法》，所以数据安全有保障 | 法律提供了框架，但**执行和合规是需要主动维护的** |

---

## 遇到这些问题怎么办？

| 你遇到的问题 | 现在可以怎么做 |
|-------------|---------------|
| 担心代码被 AI 平台保存 | 查看平台的数据使用政策。OpenAI API 默认不使用你的数据训练；网页版 ChatGPT 可以选择关闭"改进模型" |
| 想让 AI 处理敏感信息但不放心 | 用国内合规平台（如 SiliconFlow、DeepSeek、阿里云），数据不出境 |
| 想做一个收集用户数据的 AI 应用 | 在隐私政策里说明你收集什么数据、怎么用、存多久。最小化收集原则 |
| 担心 AI 模型"记住了"用户数据 | 避免在 prompt 中传入不必要的个人身份信息。用完后清除 |

---

## 你的数据去哪了？

当你用 AI 服务时，你的数据经过的路径：

```
你输入 prompt
    ↓
AI 平台服务器（处理请求）
    ↓
AI 模型生成回答
    ↓
返回给你
    ↓
（可选）平台记录对话用于模型改进
```

关键问题：**从请求到记录之间发生了什么？**

### API 模式 vs 网页版模式

| 维度 | API 调用 | 网页版聊天 |
|------|---------|-----------|
| 是否用于训练 | 默认不用于训练 | 默认可能用于训练（可手动关闭）|
| 数据留存 | 30 天后删除（OpenAI 政策） | 按平台政策 |
| 跨境传输 | 数据可能出境 | 取决于平台服务器位置 |
| 控制权 | API 用户有更多控制选项 | 有限 |

### 中国的情况

- **SiliconFlow、DeepSeek、阿里云百炼等国内平台**：数据保存在国内服务器，受中国法律管辖
- **国际平台**：数据可能出境，受当地法律管辖
- 对于实验室项目和学生项目，**选择国内平台可以规避大部分数据出境问题**

## 隐私风险一览

| 风险 | 描述 | 怎么降低 |
|------|------|---------|
| **模型记忆** | 模型可能记住训练数据中的敏感信息（如个人身份信息）| 训练数据做去标识化处理 |
| **Prompt 泄露** | 用户输入的 prompt 可能被记录或共享 | 不要输入密码、API Key 等敏感信息 |
| **输出泄露** | 模型可能输出包含敏感信息的内容 | 设置输出过滤 |
| **第三方风险** | 通过第三方平台调用 AI API 时，数据经过更多节点 | 直接使用官方 API，减少中间环节 |
| **合规风险** | 不同国家/地区对数据保护的要求不同 | 了解适用法律，选择合规平台 |

---

## 作为开发者你需要注意的

### 收集数据时

- **最小化**：只收集你真正需要的数据
- **告知**：告诉用户你收集了什么、用来做什么
- **选择**：给用户选择不让渡数据的权利
- **安全**：传输和存储时加密

### 处理数据时

- 不要在 prompt 中包含不必要的个人身份信息
- 任何 AI 生成的输出都不要直接当作"匿名数据"
- 训练自己的模型时，确保训练数据不包含敏感信息

### 上线应用时

- 写清楚隐私政策
- 提供数据删除功能
- 做好数据泄露的应急方案

---

## 你现在应该做什么？

1. 检查你常用的 AI 平台的数据使用设置——关闭"用我的数据改进模型"的选项（如果你在意的话）
2. 把你最近给 AI 发过的内容回顾一遍——有没有包含不应该发送的信息（密码、API Key、身份证号等）
3. 如果你在用 API，确认你的 API 密钥没有暴露在代码里或 GitHub 上
4. 如果你想自己做 AI 应用，先把隐私政策和数据处理流程想清楚

**要点总结**
- API 调用和网页版的隐私保护水平不同——**API 更安全**
- 国内平台的数据保存在国内，**规避了数据出境问题**
- 三个原则：**最小化收集、告知用户、提供选择**
- 不要在 prompt 中输入密码、API Key、身份证号等敏感信息
- 无论用什么平台，**检查平台的数据使用政策**——不知道就是没控制
