# 02-隐私与数据安全

> 读完本文你将了解：使用 AI 时数据的流向、核心隐私风险、如何保护自己/实验室/用户的数据，以及作为开发者/实验人员的关键注意事项

---

## 你大概率遇到过的问题
日常用 AI 调试代码、写邮件、做实验时，有没有想过这些隐私问题：
- 把实验室代码/实验数据贴给 AI 调试，这些数据会被保存吗？会被别人看到吗？
- 让 AI 写包含个人/实验室信息的邮件，这些信息会被用作什么？
- AI 平台说"用你的数据改进模型"——是不是你发的所有内容都会被用来训练？
- 想做一个 AI 应用/实验，用户/实验数据该怎么保护？出问题谁负责？

这些问题没有绝对的"是/否"答案，但核心是：**你必须知道数据的流向，以及如何主动控制风险**。

---

## 常见的误解
| 你以为的 | 实际的 |
|---------|--------|
| 用 AI 做什么平台都知道且会滥用 | 平台能看到你的输入（为了处理请求），但 API 调用默认不会用来训练（除非你手动同意） |
| AI 平台会主动保护你的数据隐私 | 隐私保护水平取决于平台服务条款 + 你的设置（比如是否开启"数据用于训练"） |
| 数据安全只是不泄露密码 | 还包括：训练数据归属、模型是否记忆敏感信息、输出是否暴露实验/用户数据 |
| 有《个人信息保护法》就万事大吉 | 法律是框架，**实验室/个人的主动合规和操作才是关键** |

---

## 遇到这些问题该怎么做？
| 你遇到的问题 | 立刻能做的解决方案 |
|-------------|---------------|
| 担心实验室代码/数据被 AI 平台保存 | 查看平台数据政策：OpenAI API 默认不用数据训练；ChatGPT 网页版可关闭"改进模型"开关 |
| 需用 AI 处理敏感信息（实验数据/个人信息） | 优先用国内合规平台（SiliconFlow、DeepSeek、阿里云百炼），数据不出境更安全 |
| 做收集用户数据的 AI 实验/应用 | 遵循"最小收集原则"，并在隐私说明里写清：收集什么、怎么用、存多久 |
| 担心 AI 模型"记住"敏感数据 | 不在 prompt 里传不必要的信息（比如身份证号、实验室机密），用完及时清理对话记录 |

## 你的数据到底去哪了？
使用 AI 服务时，数据的核心流转路径：
```
你输入 prompt（含代码/实验数据/个人信息）
    ↓
AI 平台服务器（处理请求，生成回答）
    ↓
返回结果给你
    ↓
（可选）平台记录对话/数据，用于模型改进（可手动关闭）
```

关键问题：**从"处理请求"到"记录数据"的环节，你有多少控制权？**

### API 调用 vs 网页版使用（实验室场景重点）
| 维度 | API 调用（推荐实验室使用） | 网页版聊天 |
|------|---------|-----------|
| 是否用于训练 | 默认不用于训练（可控） | 默认可能用于训练（需手动关闭） |
| 数据留存 | OpenAI 等平台 30 天后删除 | 按平台政策，留存周期不透明 |
| 跨境传输 | 可选择国内平台规避出境 | 国际平台大概率出境 |
| 控制权 | 实验人员可自定义设置 | 仅能调整基础开关 |

### 国内平台的优势（实验室/学生项目优先选）
- SiliconFlow、DeepSeek、阿里云百炼等：数据保存在国内服务器，受《个人信息保护法》管辖；
- 实验室项目/学生项目用国内平台，可规避大部分数据出境、合规风险。

---

## 核心隐私风险与规避方法
| 风险类型 | 具体描述 | 实验室场景规避方法 |
|------|------|---------|
| **模型记忆** | 模型记住训练/输入数据中的敏感信息（如实验数据、个人信息） | 实验数据做去标识化处理（隐去姓名/实验室编号）；避免用敏感数据训练模型 |
| **Prompt 泄露** | 输入的 prompt 被平台记录/共享 | 不在 prompt 里写密码、API Key、实验室机密、身份证号等 |
| **输出泄露** | 模型输出包含敏感信息（比如把实验数据直接输出） | 设置输出过滤规则；AI 生成结果后手动检查，删除敏感内容 |
| **第三方风险** | 通过第三方平台调用 AI API，数据多一次流转 | 直接用官方 API，减少中间环节（比如实验室项目别用非官方 AI 插件） |
| **合规风险** | 不同地区数据保护要求不同（比如实验数据出境） | 实验室项目优先用国内平台；了解《个人信息保护法》核心要求 |

## 实验室/开发者必须注意的原则
### 收集数据时（实验/应用场景）
- **最小化**：只收集实验/应用必需的数据（比如做用户调研，不收集无关的手机号）；
- **告知**：明确告诉用户/实验参与方：收集什么、用来做什么、保存多久；
- **选择**：给用户选择是否共享数据的权利（比如"是否同意数据用于模型优化"）；
- **加密**：数据传输（HTTPS）、存储（加密存储）全程加密。

### 处理数据时
- 不在 prompt 中包含不必要的个人/实验室身份信息；
- AI 生成的输出≠"匿名数据"，需手动检查是否含敏感信息；
- 训练自研模型时，确保训练数据不含他人敏感信息/实验室机密。

### 上线实验/应用时
- 写清晰的隐私政策（哪怕是学生项目，也要留痕）；
- 提供数据删除功能（比如用户可申请删除自己的实验数据）；
- 制定数据泄露应急预案（实验室项目至少明确"谁负责、怎么处理"）。

---

## 现在就可以做的 4 件事
1. 检查常用 AI 平台的设置：关闭"用我的数据改进模型"（实验室数据尤其要关）；
2. 回顾最近给 AI 发的内容：有没有包含密码、API Key、实验室机密等不该发的信息；
3. 检查代码/GitHub：确认 API 密钥没有硬编码、没有泄露；
4. 若做 AI 实验/应用，先写好隐私说明和数据处理流程（哪怕是简单的笔记）。

### 核心要点总结
1. API 调用比网页版更安全（实验室项目优先用 API）；
2. 国内平台规避数据出境风险，是学生/实验室项目的首选；
3. 三大核心原则：最小化收集、告知用户、提供选择；
4. 绝对不要在 prompt 中输入密码、API Key、实验室机密；
5. 关键动作：查平台政策、关数据训练、保护 API 密钥。
