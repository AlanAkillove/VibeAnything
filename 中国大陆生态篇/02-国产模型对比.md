# 02-国产模型对比

> 读完本文你将了解：DeepSeek、Qwen、Kimi、GLM 这四个国产主流模型的核心差异、每个模型擅长的场景和不擅长的场景、以及根据你的任务类型选择模型的决策框架

---

## 你可能遇到过的问题

你已经开始用国内模型做项目，但经常陷入选择困难：

- **网上全是 Benchmark 排名**，一个模型今天第一明天第三，根本没法当决策依据
- **试了 A 模型觉得不错，换到 B 发现有些地方更好**，但又说不上来到底差在哪
- **听说 DeepSeek 推理强，但用起来感觉中文不够好**——不知道是设置问题还是模型本身的问题
- **长文档处理用 Kimi，但在代码生成上质量不如预期**——怀疑是不是自己没用对
- **写一篇文章要用多个功能（长文本+代码+推理），不知道选哪个模型一次搞定**
- **想用 Qwen 因为免费，但不确定它和付费模型实际差距多大**

这些问题说明一个事实：**没有绝对的最强模型，只有最适合当前任务的那个。** 学会按场景衡量模型，比记住 Benchmark 排名有用一百倍。

---

## 常见的误解

| 你以为的 | 实际的 |
|---------|--------|
| Benchmark 高的模型在实际任务中也最好 | Benchmark 只能反映特定测试集的表现，实际任务中的体验取决于 Prompt 设计、上下文管理、模型与工具的配合 |
| 推理模型（如 DeepSeek V4 Think）什么事都比普通模型好 | 推理模型在简单任务上反而更慢更贵——写个排序算法用 Think 模式浪费，用 Flash 就够了 |
| 长上下文就是输入多长都能无损理解 | 实际使用中超过一定长度后模型会丢失细节，不同模型衰减曲线不同 |
| 所有国产模型都能做 Agent / Tool Calling | 每个模型的工具调用能力和格式要求不同，GLM 和 Qwen 在这方面比较成熟，Kimi 仍在完善 |
| 开源模型能力不如闭源模型 | Qwen-72B 和 DeepSeek V4 的开源版在很多场景不输闭源模型，开源还多了本地部署和私有化的灵活性 |

---

## 常见问题快速解决

| 你遇到的问题 | 现在可以怎么做 |
|-------------|---------------|
| 需要处理几十页的合同或论文 | 用 Kimi，它的长上下文处理能力是行业标杆 |
| 写复杂算法、做逻辑推理 | 用 DeepSeek V4 Think 或 V4 Pro，推理深度最强 |
| 做中文内容创作、翻译、润色 | Qwen 和 Kimi 都可以，Qwen 免费额度多，Kimi 长文本场景更优 |
| 开发编码 Agent 或工具 | Qwen（开源+Code Qwen CLI）或 GLM（学术+多模态），根据是否开源选 |
| 快速实验、白嫖无限调用 | Qwen 是目前免费使用最慷慨的模型 |
| 多模态场景（看图、视频理解） | Qwen-VL 和 GLM-5.2 的多模态能力较强 |
| 需要稳定 B 端服务 | Qwen（阿里百炼）或 GLM（智谱 API）的商业支持更成熟 |

## 四个模型逐一拆解

### 1. DeepSeek

**一句话定位**：国产推理王者，性价比标杆。

DeepSeek 在 2025-2026 年间的迅速崛起，靠的就是两板斧：**超强的推理能力**和**极致极低的价格**。它的 V4 系列采用 MoE 架构，实现了同等参数下更快的推理速度和更低的成本。

**核心能力分析**：

**推理和数学** ★★★★★
这是 DeepSeek 最强的领域。无论是代码逻辑追踪、数学证明、还是复杂的多步骤推理，DeepSeek 在同量级模型中处于领先水平。V4 的 Think 模式更是专门为深度推理场景优化的。

**代码生成** ★★★★☆
DeepSeek 的代码能力整体很强，尤其是算法实现和 Debug 场景。但在大型项目的结构化代码生成上，不如 Qwen-Coder 这种垂直特化模型。

**中文理解** ★★★★☆
中文能力优秀，但相比 Kimi 和 Qwen 这种纯中文友好的模型，DeepSeek 在成语、古文、方言等极端中文场景上稍逊一筹。日常使用完全没有问题。

**长上下文** ★★★★☆
V4 系列标配 1M 上下文，实际使用中前 200K 稳定，超长上下文会有性能衰减。Kimi 在长上下文领域仍是标杆。

**Agent / Tool Calling** ★★★☆☆
基础能力具备，但不如 Qwen 和 GLM 成熟。在复杂多步工具调用的场景下，稳定性有待提升。

**最适合的场景**：复杂推理、数学计算、代码 Debug、需要高性价比的通用任务。
**最不适合的场景**：超长文档分析、结构化多工具调用、对中文文化理解要求极高的文本处理。

---

### 2. Qwen（通义千问）

**一句话定位**：生态最全的国产开源模型家族。

Qwen 不是单一个模型，而是一个庞大的模型家族。从 0.5B 的超小模型到 70B+ 的专业版本，从纯文本到多模态，从通用到代码/数学/视觉特化，Qwen 的覆盖面是国产模型中最广的。

**核心能力分析**：

**代码生成** ★★★★★
Qwen-Coder 是专门为代码场景优化的特化版本，在代码补全、函数生成、跨文件修改等方面表现稳定。配合 Code Qwen CLI 使用体验很好。

**中文理解** ★★★★★
阿里在中文 NLP 上有多年积累，Qwen 的中文语料质量很高。在诗词、成语、中文语境理解上，Qwen 处于国产模型第一梯队。

**Agent / Tool Calling** ★★★★★
Qwen 在 Function Calling 和 Agent 场景的稳定性在国产模型中名列前茅。这也是为什么很多 Agent 框架（比如 LangChain 的中文社区方案）首选 Qwen 作为底层模型。

**多模态** ★★★★☆
Qwen-VL 系列在图像理解和视觉问答上表现不错。但多模态场景更新快，具体能力建议实测。

**推理和数学** ★★★★☆
Qwen-Math 是特化版本，在数学推理上表现不错。通用版本的推理能力略逊于 DeepSeek V4 Think。

**最适合的场景**：代码开发、Agent 搭建、多项能覆盖的混合任务、需要免费方案的个人项目。
**最不适合的场景**：高强度推理竞赛题、超长文档的单次处理（长上下文不如 Kimi 稳定）、需要极小体积模型部署的边缘设备（但 Qwen 0.5B 可以跑在手机上）。

---

### 3. Kimi（月之暗面）

**一句话定位**：长文本王者，中文最佳体验。

Kimi 是四家里最偏科的一个，但它的偏科方向恰好是很多用户的核心痛点：**超长上下文和中文理解**。

**核心能力分析**：

**长上下文** ★★★★★
这是 Kimi 的绝对长板。百万级别的上下文处理能力在实际场景中表现稳定：能完整理解整本小说、长篇论文、复杂合同。长文本的细节保持能力在国产模型中排第一。

**中文理解** ★★★★★
Kimi 的中文处理能力是四家里最自然的。在意译、润色、中文长文撰写场景，它的输出质量经常超出预期。中文对话的流畅度也是最好的。

**推理和数学** ★★★☆☆
推理能力不是 Kimi 的核心卖点。在复杂的数学推理、多步骤逻辑推导上，Kimi 不如 DeepSeek 和 Qwen-Math。

**代码生成** ★★★☆☆
基础代码生成够用，但在复杂的工程化代码上质量波动较大。Kimi 的强项是文档和文本，不是代码。

**Agent / Tool Calling** ★★★☆☆
仍在完善阶段。多步骤工具调用的稳定性和 Qwen 或 GLM 相比有一定差距。

**最适合的场景**：长篇文档阅读/总结、论文翻译和分析、中文内容创作、法律/金融等文档密集型工作。
**最不适合的场景**：复杂代码工程开发、高强度数学推理、需要稳定 Agent 框架的生产环境。

---

### 4. GLM（智谱 AI）

**一句话定位**：学术底蕴最深的国产模型。

GLM 系列源自清华团队，学术背景深厚。它不一定在每一项能力上都是第一名，但综合能力均衡，且在特定领域（多模态、工具调用）有自己的特色优势。

**核心能力分析**：

**多模态** ★★★★★
GLM-5.2 在视觉理解、图像描述、视频理解等多模态任务上的能力在国产模型中处于领先。如果能用到多模态场景，GLM 是非常值得一试的选择。

**Agent / Tool Calling** ★★★★★
GLM 在工具调用和 Agent 方面的设计很用心。它的 Function Calling 格式规范、稳定度高，在一些评测中甚至优于同级别的国际模型。

**中文理解** ★★★★☆
中文能力优秀，学术性文本理解尤佳——和技术论文、学术文献打交道的场景，GLM 表现很好。日常对话的流畅度略逊于 Kimi。

**代码生成** ★★★★☆
代码能力扎实，在结构化代码生成和算法实现上表现稳健。不如 Qwen-Coder 特化版，但高于 Kimi。

**推理和数学** ★★★★☆
逻辑推理能力不错，尤其是在结合多模态信息的复杂推理场景。纯文本推理略低于 DeepSeek。

**最适合的场景**：多模态应用、学术研究、教育和培训、Agent 搭建、追求均衡型模型的企业。
**最不适合的场景**：极端长文本处理（不如 Kimi）、超低成本的批量推理（不如 DeepSeek 的 MoE 提效省钱）、纯中文创作（不如 Kimi 自然）。

## 按场景选择的决策框架

`
你的核心任务是什么？
├── 复杂推理/数学/算法
│   └── DeepSeek（V4 Think 首选，V4 Flash 快速备选）
├── 代码开发
│   ├── 通用代码 → Qwen-Coder（特化最强）
│   └── 代码 Debug/优化 → DeepSeek（推理深）
├── 长文本处理（论文/合同/文档）
│   └── Kimi（长上下文标杆）
├── 中文内容创作
│   ├── 长篇创作 → Kimi（自然流畅）
│   └── 短篇/多轮 → Qwen（性价比高）
├── 多模态（图像/视频理解）
│   └── GLM（多模态最强）
├── Agent / 工具调用
│   ├── 生产环境 → Qwen（最稳定）
│   └── 多模态 Agent → GLM
├── 学术研究
│   └── GLM（学术底蕴）或 DeepSeek（推理辅助）
├── 低成本/高并发
│   ├── DeepSeek V4 Flash（MoE 省成本）
│   └── Qwen（免费版本多）
└── 综合全能（什么都做一点）
    └── Qwen 或 DeepSeek（覆盖面最广，按场景切换版本）
`

## 要点总结

1. **DeepSeek = 推理之王**：MoE 架构带来极致性价比，V4 Think 推理深度最强，V4 Flash 日常够用且便宜
2. **Qwen = 全能选手**：模型家族最全面（0.5B-70B+），代码特化和 Agent 能力突出，免费额度慷慨
3. **Kimi = 长文本之王**：百万级上下文处理行业标杆，中文理解和创作自然流畅，代码和推理是弱项
4. **GLM = 学术多面手**：学术根基最深，多模态和 Agent/工具调用能力突出，综合能力均衡
5. **没有万能模型**：成熟开发者会按任务切换——推理用 DeepSeek，代码用 Qwen，长文档用 Kimi，多模态用 GLM
6. **不要只看 Benchmark**：每个模型的实际体验和你的 Prompt 质量、工具搭配、任务类型高度相关，自己试最重要
