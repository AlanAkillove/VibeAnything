# 03-API获取与成本

> 读完本文你将了解：国内 AI API 的核心获取平台和注册流程、各平台的免费额度和付费价格对比、支付方式和预算控制方法、以及 Kimi 双平台这个容易踩坑的特殊问题

---

## 你可能遇到过的问题

你已经知道要用 API 而不是网页版来开发项目，但在实际操作中遇到了各种门槛：

- **注册了好几个平台**，每个都要填信息、实名认证、绑定支付——流程走完就累了
- **免费额度用完后自动扣费**，月底收到账单才发现花了几百块——事前完全没预算概念
- **在某个平台充了值，发现不支持切换模型**——钱被锁定在一个平台了
- **看到官方的模型价格表头大**——输入输出价格不同、缓存价格不同、不同模型各自定价
- **注册了 Kimi 发现登录不了**——在 .com 充的值，但 SDK 连的是 .ai，两边账号不互通
- **不知道怎么控制预算**——只想每个月花 50 块，但平台没有月度限额功能，得自己手动停

这些问题的核心是：**API 平台的选择直接影响你的开发成本和体验**。选对平台（和支付方式）比选对模型更先发生。

---

## 常见的误解

| 你以为的 | 实际的 |
|---------|--------|
| 充了值就能用所有模型 | 每个平台只覆盖自己的模型或合作的第三方模型，硅基流动覆盖面最全，但旗舰模型可能不包含 |
| 免费额度用完就扣费 | 大部分平台默认开启按量计费，关闭需要手动操作或设置预算告警，新手容易被溢出收费 |
| 一定得实名认证才能用 | 硅基流动的基础使用不需要实名，但部分免费模型需要；DeepSeek 注册不实名也有额度 |
| API 平台的模型和网页版是同一个 | 不一定，某些平台网页版用的是独立版本（如限定了上下文长度），API 版能力更完整 |
| 所有国内平台都兼容 OpenAI SDK | 大部分兼容，但支付宝/微信的支付方式、实名流程、模型命名规则各不相同 |

---

## 常见问题快速解决

| 你遇到的问题 | 现在可以怎么做 |
|-------------|---------------|
| 不想每个平台都注册一遍 | 只用 SiliconFlow 一个账号就能玩大部分开源模型 |
| 担心被意外扣费 | 首次充值 10 元测试，设置平台的月度消费告警，不用时把 API Key 停用 |
| 想先不花钱试试 | SiliconFlow 有免费额度，DeepSeek 注册赠送体验额度 |
| 学生没有信用卡 | 国内平台全支持支付宝，微信也可以用于某些平台 |
| 发现在已经充值的平台不能换模型 | 优先选模型覆盖广的平台（如 SiliconFlow），不要把预算分散在小平台 |
| Kimi 两边分不清 | 看下一节的详细解释，这个问题已经让很多开发者白花钱了 |

## 四大平台逐一拆解

### 1. SiliconFlow（硅基流动）—— 新手首选，模型超市

**一句话**：一个账号玩所有，最适合入门。

**注册流程**：
1. 打开 cloud.siliconflow.cn
2. 手机号注册，60 秒完成
3. 无需实名认证即可使用基础功能
4. 部分免费模型需要实名，按提示上传身份证即可

**免费额度**：
- 新用户注册赠送 14 元体验金
- 不定期的活动赠送额度（关注官网公告）
- 部分低价模型的免费调用额度

**费用结构**（以常见模型为例，具体价格以控制台为准）：
- Qwen 系列：输入约 1-4 元/百万 Token，输出约 2-12 元/百万 Token
- DeepSeek V4 Flash：输入约 0.5 元/百万 Token，输出约 2 元/百万 Token——极低成本首选
- GLM-5.2：输入约 2 元/百万 Token，输出约 8 元/百万 Token

**支付方式**：支付宝（网页扫码，支持余额充值）
**月度预算建议**：个人学习项目 20-50 元/月足够，重度使用 100-200 元/月

**优势和注意点**：
- 模型覆盖最广：上百款模型，一个 API Key 搞定全部
- OpenAI 兼容接口：切换模型只需要改 model 名字，代码不用动
- 国内网络友好：无需任何特殊网络配置
- 部分旗舰模型时效性稍有延迟（大厂优先自家平台更新）
- 模型数量和平台稳定性之间需要平衡

**使用建议**：
```python
from openai import OpenAI

client = OpenAI(
    api_key="sk-你的硅基流动 API Key",
    base_url="https://api.siliconflow.cn/v1"
)

# 只需改 model 名字就能切换任意模型
response = client.chat.completions.create(
    model="Qwen/Qwen3.7-72B-Instruct",  # 换成 deepseek-ai/DeepSeek-V4 或者 THUDM/GLM-5.2 都可以
    messages=[{"role": "user", "content": "你好"}]
)
```

### 2. DeepSeek Platform —— 推理首选，极致性价比

**一句话**：如果你确定主要用 DeepSeek 模型，直接走官方平台。

**注册流程**：
1. 打开 platform.deepseek.com
2. 邮箱或手机号注册
3. 新账号赠送体验额度
4. 使用前在控制台左侧 API Keys 页面创建密钥

**免费额度**：
- 注册赠送一定额度的免费体验 Token（具体以当前活动为准）
- 无长期免费额度，用完即按量计费

**费用结构**（DeepSeek 自家平台的优势在于提示缓存机制可大幅降本）：
- V4 Flash：输入极低、输出也低，配合提示缓存可做到行业最低成本
- V4 Pro：旗舰版，价格略高但推理能力更强
- V4 Think 模式：推理输出 Token 更长，按 Token 计费

**支付方式**：支付宝（支持充值，也可以绑定后按量扣费）
**月度预算建议**：20-50 元/月足够中轻度使用

**优势和注意点**：
- 直接使用最新版本，无需等待第三方平台更新
- 提示缓存机制：重复上下文场景可节省 90% 输入成本
- 1M 上下文窗口
- 只覆盖 DeepSeek 自家模型，无法切换其他模型
- 海外平台，注册流程比国内平台稍慢（但支持支付宝是独特优势）

### 3. 阿里云百炼 —— 企业首选，全能生态

**一句话**：如果你已经用阿里云，或者需要 B 端服务，百炼是最佳选择。

**注册流程**：
1. 打开 bailian.aliyun.com
2. 阿里云账号登录（如果没有先注册阿里云账号）
3. 开通百炼服务
4. 实名认证（国内云服务的标准流程）
5. 在控制台创建 API Key

**免费额度**：
- 新用户 Qwen 系列有一定量的免费调用额度
- 不定期活动赠送
- 部分基础模型长期免费

**费用结构**（按模型版本和规格定价）：
- Qwen-Turbo：低成本版本，适合轻量任务
- Qwen-Plus：均衡版本，适合大多数场景
- Qwen-Max：最强版本，适合高难度任务
- 具体价格在百炼控制台的模型广场查看

**支付方式**：支付宝、微信、对公转账（企业用户）
**月度预算建议**：个人 30-100 元/月，企业按实际调用量评估

**优势和注意点**：
- 与阿里云服务无缝集成（函数计算、OSS、ECS）
- B 端服务成熟，有技术支持和 SLA
- 模型训练、微调、部署全套能力
- 支持多种支付方式，企业可走合同流程
- 注册流程长，需要实名认证
- 模型选择不如 SiliconFlow 多样化（主要是 Qwen 系列和合作模型）

### 4. Kimi Platform —— 长文首选，注意双平台陷阱

**一句话**：长文本场景的最佳 API 选择，但有个容易踩坑的特殊问题。

**注册流程**：
1. 打开 Kimi 开放平台，注意有两个入口：
   - **platform.kimi.com** —— 面向国内用户
   - **platform.moonshot.ai** —— 面向国际用户
2. 两个平台的账号、额度、价格完全不同，**不互通**
3. 国内用户建议用 platform.kimi.com（支持支付宝）
4. 邮箱或手机号注册

**免费额度**：
- 新用户赠送体验额度

**费用结构**（以官方公布为准）：
- 按 Token 计费，输入和输出价格不同
- 长上下文场景有额外计费逻辑（处理更大的输入上下文会产生更高的费用）

**支付方式**：platform.kimi.com 支持支付宝；moonshot.ai 需要国际信用卡
**月度预算建议**：长文本场景 50-100 元/月（输入 Token 多、上下文长，消耗会比轻量任务高）

**重要：Kimi 双平台的陷阱**

这是 Kimi API 用户最容易踩的坑。Kimi 有两个独立的 API 平台：

platform.kimi.com（国内用户使用，推荐）：
- 支持支付宝充值
- 国内网络直接访问
- 手机号/邮箱注册
- 和 Kimi 网页版账号关联

platform.moonshot.ai（国际用户使用）：
- 需要国际信用卡
- 在国内可能不稳定
- 邮箱注册
- 和 kimi.com 账号独立

**常见问题**：
- 你在 kimi.com 注册、充了值，却用 moonshot.ai 的 base_url 调用——报错说余额不足
- 你在 moonshot.ai 也注册了，两边充两次钱，两边都有余额但两边都不够用
- 忘记自己是哪个平台注册的，需要反复确认

**解决方案**：
- 统一在 platform.kimi.com 注册和充值
- SDK 中 base_url 设置为 https://api.kimi.com/v1（注意不是 moonshot.ai）
- 如果不确定，查阅控制台页面的接入文档，里面有正确的 endpoint 信息

## 预算控制策略

**新手最实用的预算控制方法**：

**第一步：从免费额度开始**
选择有免费额度的平台（SiliconFlow 或 DeepSeek），先跑通代码、验证效果，确定你的调用量之后再说付费。

**第二步：小额度测试**
确定要付费后，先充值 10-50 元。这个量级够你运行几百到几千次基础调用，足够评估是否真的需要。

**第三步：设置消费告警**
各平台的告警设置入口：
- SiliconFlow：控制台 -> 费用管理 -> 设置月度消费上限
- DeepSeek：控制台 -> 用量管理 -> 设置用量告警
- 阿里云百炼：控制台 -> 费用中心 -> 设置预算告警

**第四步：定期检查用量**
养成每周查看一次控制台用量页面的习惯。如果发现异常（调用量突然暴增），第一时间吊销 API Key 并重新生成。

## 平台选择决策表

| 你的情况 | 推荐平台 | 理由 |
|---------|---------|------|
| 新手入门、对比模型 | SiliconFlow | 一个账号玩所有，免费额度试用 |
| 主力 DeepSeek 模型 | DeepSeek Platform | 最新版本、提示缓存省钱 |
| 企业项目、需要支持 | 阿里云百炼 | B 端成熟、生态齐全 |
| 长文档处理为主 | Kimi Platform | 长上下文能力最强 |
| 多模型混合使用 | SiliconFlow + 特定平台 | 通用场景用硅基，特殊模型走官方 |
| 学生、预算极低 | SiliconFlow 免费额度+DeepSeek Flash | 两平台免费/低价组合，成本可控 |

## 要点总结

1. **SiliconFlow = 新手最优解**：一个账号玩全部开源模型，注册快、有免费额度、支持支付宝
2. **DeepSeek Platform = 低成本推理最佳选择**：提示缓存机制是省钱利器，极低成本的运行主流模型
3. **阿里云百炼 = 企业首选**：生态集成好、B 端服务成熟、支持企业支付方式
4. **Kimi Platform = 长文本场景唯一选择**：长上下文能力无可替代，但要警惕 .com 和 .ai 双平台不互通的陷阱
5. **预算控制四步法**：免费起步 -> 小额度测试 -> 设置告警 -> 养成检查习惯
6. **不要把所有预算押在一个平台**：保持 2-3 个平台有余额，灵活应对不同场景需求
