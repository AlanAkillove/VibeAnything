# 01-国内AI工具全景

> 读完本文你将了解：2026 年中国 AI 工具生态的三大阵营分布、大厂系/创业系/平台系各自的代表产品和定位、它们之间的合作关系与竞争格局、以及你怎么根据自己的需求选择合适的阵营

---

## 你可能遇到过的问题

你打开公众号或技术社区，看到各种国产 AI 模型的名字铺面而来——通义千问、混元、文心、豆包、DeepSeek、Kimi、GLM……广告文案都说自己遥遥领先，但你真的想搞清楚时，却发现信息杂乱、概念混淆：

- **这些模型之间到底是什么关系？**——有些同属一个阵营，有些是直接竞争对手，还有些是上下游合作
- **阿里自己就有 Qwen，为什么 SiliconFlow 上也能用 Qwen？**——这些平台之间的关系让人一头雾水
- **哪些是免费的直接能用，哪些要花钱申请 API？**——网页版和 API 两种入口的差异没弄清楚，先入了个正版大坑
- **听到创业系的公司名字感觉很小，不放心**——不知道它们的实际能力和规模
- **大厂的东西肯定更好用吧？**——不一定，创业系在某些能力上甚至更强

这背后其实是一张清晰的生态版图。认清这个版图，你才知道选谁、怎么选、为什么这么选。

---

## 常见的误解

| 你以为的 | 实际的 |
|---------|--------|
| 国产 AI 只有百度文心一家 | 2026 年中国 AI 工具生态至少有几十个模型在激烈竞争，呈现大厂+创业+平台三足鼎立 |
| 大厂模型一定比创业系强 | 大厂有资源，创业系更灵活——DeepSeek 在推理能力上长期领先，Kimi 的长上下文处理是行业标杆 |
| SiliconFlow 是某个大厂的产品 | 它是独立的第三方平台，核心业务就是托管各家模型并提供 API 服务，像个 AI 模型超市 |
| 字节的豆包只是聊天机器人 | 豆包 2.0 系列涵盖旗舰/轻量/代码多版本，底层模型在编码和 Agent 场景表现不俗，有 API 接入方式 |
| 同阵营的模型能力都差不多 | 即使是同一家大厂，不同版本定位差异很大（比如 Qwen 从 0.5B 到 70B+，每个型号用途不同） |
| 我只要用一个模型就够了 | 实际开发中，不同任务用不同模型是常见做法——长文本用 Kimi、推理用 DeepSeek、代码用 Qwen、快速响应用 Flash 系列 |

## 常见问题快速解决

| 你遇到的问题 | 现在可以怎么做 |
|-------------|---------------|
| 想快速试玩各种国产模型，不想一个个注册 | 直接去 SiliconFlow 注册一个账号，里面集成了几乎所有国产开源模型 |
| 网页版聊着不错，想调 API 来用 | 确认该模型是否开放 API（大厂大多提供，部分免费），参考 03 篇了解申请方式 |
| 想给公司选型，评估国产模型的稳定性 | 大厂系在 B 端服务上最成熟（有 SLA、技术响应用人的支持）；创业系建议走平台托管 |
| 分不清哪些是开源、哪些是闭源 | Qwen/GLM/DeepSeek/MiniMax M3/MiMo/Step 都开源——混元 Hy3 是腾讯首次开源，文心和豆包仍闭源仅通过自家 API 使用 |
| 想知道哪个模型最新、最强 | 关注各家的官方公众号或知乎号，不要看第三方榜单（很多有水分） |

## 三大阵营全景

### 第一阵营：大厂系（自有模型+云服务）

这是中国 AI 界最有钱有人的阵营。大厂系的共同特点是：**模型覆盖范围广、云生态绑定深、B 端服务成熟**。

**阿里（通义千问 / Qwen 系列 / 百炼平台）**

阿里在 AI 上的投入密度在国内排在最前面。Qwen 系列从 0.5B 到 70B+ 的参数覆盖是全行业最全的。核心产品线包括：

- **Qwen 模型系列**：包括多模态（Qwen-VL）、代码专用（Qwen-Coder）、数学推理（Qwen-Math）等特化版本——开源，社区活跃度高
- **百炼平台（bailian.aliyun.com）**：阿里云的 AI 开发平台，统一的 API 入口，提供模型训练、推理部署、RAG 搭建等全套服务
- **通义千问网页端**：面向 C 端用户的聊天产品
- **Code Qwen CLI**：终端编码 Agent，完全免费

阿里云百炼是 B 端用户的首选之一，因为它和阿里云的其他服务（函数计算、OSS、ECS）可以无缝集成。

**腾讯（混元大模型）**

腾讯混元的策略比较稳重，但 2026 年迎来了重要转变。核心动态：

- **混元大模型（闭源）**：腾讯自研底座模型，通过腾讯云 API 输出
- **Hunyuan Hy3 Preview（2026 年 4 月）**：腾讯首次开源大型语言模型，295B 参数（21B 激活），MoE 架构，支持 256K 上下文。在 SWE-Bench 等编码和 Agent 基准测试中表现突出，尤其擅长数百步的长流程 Agent 任务
- **Hunyuan Video**：在视频生成领域表现突出，是国内视频生成的代表模型之一
- **内部应用广泛**：混元已深度嵌入微信、QQ、腾讯文档、腾讯会议等产品
- **企业服务成熟**：通过腾讯云的 TI-ONE 平台提供模型服务和私有化部署

对个人开发者来说，腾讯混元的入口在逐渐开放。Hy3 开源后可通过 SiliconFlow 等平台调用。

**百度（文心大模型 / 千帆平台）**

百度是中国最早布局大模型的互联网公司之一。文心大模型经过多次迭代，形成了完整的产品矩阵：

- **文心一言**：C 端对话产品，可以直接使用
- **千帆大模型平台**：百度智能云的 B 端 AI 开发平台，和阿里百炼类似
- **文心 API**：提供标准化的模型调用接口
- **搜索生态协同**：文心已经和百度搜索深度整合

百度的优势在于搜索数据和中文语料的积累，文心在中文理解上有自然优势。

**字节跳动（豆包 / 火山引擎）**

字节跳动是 2026 年中国 AI 市场上用户感知度最高的公司之一——豆包 App 的月活跃用户以亿计，已经成为很多人日常使用 AI 的第一入口。2026 年 2 月发布的 **豆包 2.0** 系列将能力推到了新高度：

- **豆包 2.0 Pro**：旗舰模型，在多项基准测试中达到国内顶尖水平
- **豆包 2.0 Lite / Mini**：轻量版本，Mini 可在端侧运行
- **豆包 2.0 Code**：代码特化版本，编程能力显著提升
- **OS Agent**：操作系统级 Agent 产品，可自主操作手机界面完成任务
- **实时语音 3.0 API（2026 年 6 月）**：端到端语音对话，延迟极低

在 B 端，字节通过 **火山引擎** 提供模型 API 和训练基础设施。**扣子（coze.cn）** 是字节的 AI Bot 搭建平台，类似国外的 GPTs。

如果你是 C 端用户，豆包 App 是最直接的产品；如果你是开发者，火山引擎或 SiliconFlow 都可调用豆包模型。

**小米（MiMo 系列）**

小米在 AI 大模型领域后来居上。2026 年 4 月发布的 **MiMo-V2.5** 展现了在长文本和 Agent 方向的独特定位：

- **MiMo-V2.5**：支持 1M 上下文，开源，在长文本理解和代码生成上表现稳健
- **MiMo Claw**：AI 编程助手，定位类似 Codex CLI，面向开发者
- **UltraSpeed**：推理加速引擎，可达 1000+ TPS 生成速度
- **端侧优势**：依托小米的手机和 IoT 生态，在端侧推理部署上有天然场景

MiMo 的 API 主要通过小米自身平台输出，开发者生态仍在建设中。

### 第二阵营：创业系（技术驱动+差异化竞争）

这是中国 AI 最骚气的阵营——没有大厂的包袱，靠技术创新打出差异化。

**DeepSeek（深度求索）**

DeepSeek 是 2025-2026 年最受关注的国产 AI 公司，以其超强推理能力和极致的性价比出圈。

- **核心优势**：推理能力对标甚至超越同参数量级的国际模型、支持 1M 超长上下文、提示缓存机制大幅降低重复输入成本
- **产品线**：DeepSeek V4 系列（旗舰/Flash 双版本，内置 Think 推理模式）
- **商业模式**：API 按量付费，价格极具竞争力；全部开源
- **Mixture of Experts（MoE）架构**：同参数量级下推理速度更快、成本更低

DeepSeek 在开发者圈子里口碑很好，是很多人的第二模型选择。

**MiniMax（稀宇科技）**

MiniMax 是 2026 年最让人意外的国产黑马。2026 年 6 月发布的 **M3** 一举进入全球开源模型第一梯队：

- **M3 模型**：428B 参数（23B 激活），MoE 架构，开源，支持 1M 上下文
- **惊人表现**：SWE-Bench 编码得分超过 GPT-5.5，BrowseComp 达 83.5——编码和 Agent 能力极强
- **Hailuo AI**：视频生成产品，在国内视频生成领域占有一席之地
- **API 服务**：通过 MiniMax 官方平台或 SiliconFlow 调用

M3 的开源发布是 2026 年 6 月中国 AI 圈最受关注的事件之一——它的编码能力让很多开发者重新评估"国产模型能否用于生产级编程"。

**月之暗面（Kimi / Moonshot AI）**

Kimi 以超长上下文处理能力出名，是国内最早主打长文本的 AI 产品。

- **核心优势**：超长上下文处理（百万 token 级别）、中文长文本理解和总结能力出色
- **产品线**：Kimi 网页/App 端、Kimi Code CLI、Kimi API（注意 .ai 和 .com 双平台不互通）
- **目标用户**：知识工作者、研究人员、需要大量文档处理的用户
- **差异化**：长文档翻译、论文阅读、合同审查等场景处于领先

Kimi 的网页产品体验很好，API 方面仍在快速发展。

**智谱 AI（GLM 系列 / Zhipu AI）**

智谱 AI 源自清华大学的技术团队，GLM 系列是国产大模型中学术根基最深的一家。

- **核心优势**：学术背景深厚、GLM-5.2 在多项基准测试中表现出色、GLM 开源生态成熟
- **产品线**：智谱清言（C 端产品）、GLM API（B 端）、模型定制服务
- **开源贡献**：GLM 系列从早期就坚持开源路线，是国产模型开源的重要推动力
- **多模态能力**：GLM-5.2 在视觉理解、Agent 工具调用方面表现不俗

智谱是很多学术研究和教育机构的首选。

**阶跃星辰（Step）**

阶跃星辰是快速崛起的新锐力量，其 **Step 系列** 以 Agent 场景优化为核心理念：

- **Step 3.7 Flash（2026 年 5 月）**：196B 参数（11B 激活），MoE 架构，Apache 2.0 开源，推理速度达 400 TPS
- **Agent 优化**：专门针对工具调用和 Agent 执行场景设计，Function Calling 稳定性突出
- **超低延迟**：400 TPS 推理速度，适合对响应速度要求高的实时场景
- **SiliconFlow 可调用**：已上架 SiliconFlow，开发者可方便测试和使用

Step 的差异化在于"不做最大的模型，做最快的 Agent 模型"。如果你的场景是高频工具调用或实时 Agent 应用，Step 是值得关注的选择。

### 第三阵营：平台系（模型聚合+API 代理）

这是 2025-2026 年兴起的新模式——不做模型，专注做模型超市。

**SiliconFlow（硅基流动）**

SiliconFlow 是目前国内最大的开源模型托管平台之一，对个人开发者非常友好。

- **核心定位**：第三方模型托管平台，集成数十个开源模型
- **覆盖模型**：Qwen、DeepSeek、GLM、Llama、Yi、Phi、Step 等上百款
- **平台特点**：统一 API 接口（兼容 OpenAI 格式）、支持支付宝、有免费额度
- **适合人群**：想用多款模型做对比的新手、不想为每个模型单独注册 API 的用户

SiliconFlow 不自己训练模型，它的价值在于聚合和托管——降低了使用门槛和切换成本。

## 阵营之间的关系

大厂和创业系是直接竞争关系，但各有侧重——大厂靠生态和云服务锁定用户，创业系靠技术亮点吸引开发者。平台系则与二者合作：大厂和创业系的开源模型通过 SiliconFlow 触达更多开发者，平台借这些模型丰富自己的产品线。

**关键认知**：大厂内部也不完全互通。阿里 Qwen 和腾讯混元互不兼容，API 调用的 SDK 和计费方式各自独立。这意味着你无法用一个账号通吃所有大厂模型。

## 怎么选阵营：决策建议

| 你的场景 | 推荐阵营 | 理由 |
|---------|---------|------|
| 入门学习、对比测试 | 平台系（SiliconFlow）| 一个账号玩所有模型，免费额度够用 |
| 中小型项目、API 接入 | 创业系（DeepSeek/MiniMax/Step） | 性价比高、文档清晰、社区活跃 |
| 大型企业应用 | 大厂系（阿里百炼/百度千帆）| B 端服务完善、有 SLA 支持、生态集成好 |
| 学术研究、教育 | 创业系（GLM 智谱） | 学术背景深、开源生态好、适合深入理解 |
| 快速原型、个人工具 | 平台系 + 创业系混用 | 原型用免费平台，上线转稳定 API |
| 长文档处理 | 创业系（Kimi） | 长上下文能力最优 |
| 编码 Agent 开发 | 创业系（MiniMax M3 / Step）/ 大厂系（豆包 Code） | 编码和 Agent 场景各模型发力，建议多测 |

## 要点总结

1. **三大阵营界限清晰**：大厂系（钱多生态广）、创业系（技术驱动差异化）、平台系（聚合托管降低门槛）
2. **大厂系看五家**：阿里 Qwen（生态最全）、字节豆包（用户最广）、腾讯混元（Hy3 开源转型）、小米 MiMo（端侧优势）、百度文心（搜索协同）
3. **创业系看五家**：DeepSeek（推理+性价比）、MiniMax（M3 开源编码黑马）、Kimi（长文本+中文理解）、GLM（学术积淀+多模态）、阶跃星辰（Agent 优化+超低延迟）
4. **平台系认准 SiliconFlow**：一个账号玩所有开源模型，国内网络友好、支持支付宝、有免费额度
5. **阵营之间是竞合关系**：大厂和创业系竞争，但开源模型通过平台系触达更多开发者。2026 年出现了"大厂也开始开源"的新趋势（腾讯 Hy3、小米 MiMo）
6. **不选边站队**：成熟的开发者往往混用——原型用平台系、工程用大厂系、特殊场景用创业系
