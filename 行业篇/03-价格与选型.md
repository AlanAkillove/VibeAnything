# 03-价格与选型

> 读完本文你将了解：大模型定价的主要模式、不同模型之间的相对价格定位、怎么估算项目的 AI 成本、以及根据成本和质量做决策的框架

---

## 你可能遇到过的问题

你在尝试把 AI 集成到自己的项目里，但一涉及到钱就变得不确定了：

- **用 API 按 token 计费，完全不知道一个月要用多少**——写一个 demo 花了 2 块钱，感觉不贵，但到了生产环境几百个用户同时用，月底账单会不会吓死人？
- **看到各种定价方案完全算不明白**——输入和输出价格不一样、缓存有折扣、batch 有折扣，还有按时间包月、按 token 包量……比手机套餐还复杂
- **免费额度用完就不知道怎么选了**——各家都有免费额度，用起来很爽，但真正要掏钱时各家价格差很多，不知道怎么比
- **听说 DeepSeek 很便宜，GPT 很贵**——但具体贵多少、便宜多少，没有概念，不知道贵出来的那部分值不值得
- **码力全开的模型和轻量模型混在一起**——同一个厂商的模型有高端版、性价比版、极速版，不同版本价格差很多

这些问题的根源：**大模型的定价体系比普通 SaaS 复杂得多**，而且没有统一标准。不懂定价结构就没办法估算成本，而成本估算不准，项目就可能做不下去。

---

## 常见的误解

| 你以为的 | 实际的 |
|---------|--------|
| 模型定价只看"每个 token 多少钱" | 定价变量比这多得多——输入 vs 输出价格不同、缓存价格不同、批量价格不同、不同时段的折扣也不同 |
| 便宜的模型做出来的东西质量一定差 | 2026 年的现状是性价比模型（如 DeepSeek Flash、Qwen 轻量版）在大多数常规任务上质量已经相当好，只有复杂任务才需要旗舰版 |
| 价格贵=更聪明，全方位更好 | 高端模型确实在复杂推理上更强，但在简单任务上和便宜模型没有区别，用贵模型做简单任务是在浪费钱 |
| 免费的开源模型 = 零成本 | 文章 02 已经讲过——开源模型本地部署有 GPU、运维、电力等隐性成本，量大的话自建划算，量小的话 API 反而更便宜 |
| 定价是固定的，签了合同就不变了 | 2026 年大模型定价仍在快速下降，去年比今年普遍贵不少——按年签合同要考虑降价预期 |

---

## 常见问题快速解决

| 你遇到的问题 | 现在可以怎么做 |
|-------------|---------------|
| 想估算一个项目的 AI 成本 | 先用一个估算器：预估日均请求量 x 平均输入 token 数 x 平均输出 token 数 x 价格，算出月消耗；再上浮 30% 留余量 |
| 不知道用哪个档次 的模型 | 先用旗舰版跑一周，看实际效果；然后换性价比版跑同样的任务，对比质量差异——如果差别不大，就切到性价比版 |
| 预算有限但需要好质量 | 用混合策略：80% 的请求走便宜模型（日常问答、简单分类），20% 的高难度请求走旗舰版（复杂推理、代码审查） |
| 需要处理大量数据但成本敏感 | 使用 batch API（离线批处理），通常价格是实时 API 的一半左右；同时开启 prompt caching，重复内容大量省钱 |
| 担心账单失控 | 几乎所有平台都支持用量上限（usage limit）和预算告警——设定月预算上限，到 80% 时自动预警 |

## 大模型的定价模式

2026 年的大模型定价有以下几种主流模式，理解它们才能算明白账。

### 1. 按 Token 计费（最常见）

这是最基本的模式：根据你使用的 token 数量收费。

关键变量：
- **输入价格 vs 输出价格**：输出 token 通常比输入贵 3-4 倍，因为生成的计算量更大
- **上下文长度阶梯**：长上下文输入可能额外计费（消耗更多算力），部分厂商按输入长度分档收费
- **缓存价格**：prompt caching 命中的输入 token 价格大幅降低（通常打 4-5 折），适合系统提示词固定的场景

### 2. 按请求计费 + Token 叠加

部分模型有基础请求费，再加 token 费。这种模式适合短对话场景（每次请求 token 少但请求次数多），或者需要固定成本的场景。

### 3. 订阅制 / 包月制

适用于高频个人用户或小团队。付固定月费获得一定量的使用额度，超出后降速或按量计费。适合使用量稳定的场景，缺点是用不完的额度不累积。

### 4. 批处理 / Batch API

将非实时任务批量提交，价格通常是实时 API 的一半或更低。适合"今天提交、明天取结果"的场景（大量数据处理、夜间批量推理）。

### 5. 免费额度

几乎所有国产闭源模型都提供可观的免费额度，适合学习、个人项目、小规模原型验证。但免费额度通常有速率限制（RPM/TPM），生产环境不够用。

## 价格梯队定位

以下按**相对价格水平**对模型进行分类（不涉及具体数字，2026 年中市场行情）：

| 价格梯队 | 代表模型 | 特征 |
|---------|---------|------|
| **旗舰级** | GPT 最新版、Claude 高端版、Gemini 最新版 | 价格最高，但能力最前沿。适合对质量极度敏感的场景，比如复杂代码审查、关键业务决策辅助 |
| **主流级** | Claude 标准版、DeepSeek 旗舰版、GLM 闭源版 | 价格适中，综合能力强。适合绝大多数生产场景，是大多数团队的主力模型 |
| **性价比级** | DeepSeek Flash / Lite、Qwen 中等版、Kimi 标准版 | 价格显著低于旗舰级，大部分任务质量和旗舰级差距不大。适合成本敏感但质量不能太差的项目 |
| **入门级 / 免费级** | Qwen 轻量版、国产模型免费额度、开源模型自托管 | 成本极低或免费，能力足够应对简单任务。适合学习、原型、简单分类和抽取场景 |

**重要提示**：同级别模型的价格在 2026 年下降趋势明显，每 3-6 个月就可能有一次调价。做预算时建议用**当前价格的 80%** 做长期规划，因为降价概率大于涨价概率。

## 影响成本的五个关键变量

选型时不只看模型单价，还要看这些因素：

### 1. 输入输出比例

不同任务类型的输入输出比差异很大：
- **分类 / 抽取任务**：输入很长（文档），输出很短（几个字），输出成本占比低
- **生成 / 创作任务**：提示词很短，输出很长（文章、代码），输出成本占比高
- **对话任务**：输入输出都适中，且随着轮数增加上下文不断变长

### 2. Prompt 长度与缓存命中率

如果系统提示词很长（几千 token）但内容固定，prompt caching 能把这部分成本降到很低。设计 prompt 时把**固定部分（系统提示）和可变部分（用户输入）拆开**，能让缓存命中率最大化。

### 3. 并发需求

高并发场景下，价格不是唯一成本——延迟和失败率同样影响用户体验。可能需要选择**推理速度更快、吞吐更高**的模型，即使它单价稍高。

### 4. 模型切换成本

频繁切换模型不仅仅是改接口地址的问题：不同模型对 prompt 风格敏感度不同，换模型可能需要重新调 prompt。这部分人力成本也是总成本的一部分。

### 5. 合规与部署成本

如果选择本地部署开源模型，除了 GPU 硬件和电力，还需要考虑：
- 数据中心的机架和网络
- AI infra 工程师的薪资
- 模型更新的持续投入

这些隐性成本经常被低估。一个简单的换算：**如果你的月 API 账单低于 2000-5000 元人民币，自建大概率不划算**。

| 项目规模 | 预算 | 质量要求 | 首选方案 |
|---------|------|---------|---------|
| 个人学习 / 实验 | 极低 | 能用就行 | 各家免费额度 + Qwen 轻量版 |
| 个人项目 / 小工具 | 低 | 一般 | DeepSeek 性价比版 / 国产模型免费后付费 |
| 创业公司 MVP | 中 | 较高 | 闭源 API 起步，后期混合 |
| 生产环境 AI 功能 | 中高 | 高质量 | 旗舰 + 性价比混合策略 |
| 企业核心业务 | 高 | 极高+合规 | 开源自建为主 + 闭源兜底 |

## 长期成本策略

**不要为 100% 的完美付 10 倍的价格。** 模型能力的边际收益递减很明显：旗舰版可能比性价比版贵 5-10 倍，但在你的任务上可能只好了 10-20%。选型的艺术在于找到**性价比拐点**——多付一点钱能换取明显质量提升的那个区间。

### 实战建议

1. **先跑通再优化**：MVP 阶段不要太多考虑成本，用闭源 API 快速验证价值
2. **设置用量告警**：所有平台都有额度设置功能，上线前务必设好月预算上线
3. **定期回顾账单**：每季度检查一次模型使用量和费用，看有没有优化空间（切模型、开缓存、加 batch）
4. **关注降价动态**：2026 年大模型价格仍在下降通道，每半年重新评估一次定价策略
5. **预留换模型的空间**：代码层做接口抽象，不要把某个厂商的特性耦合太深

## 要点总结

1. **大模型定价体系很复杂**：输入 vs 输出、缓存、batch、订阅、免费额度——要算清楚需要理解这些变量
2. **选模型不是越贵越好**：性价比模型在大多数任务上已经足够好，旗舰版只在对质量极度敏感的场景才值得多花钱
3. **总成本 = API 费 + 人力 + 基础设施**：本地部署开源模型有隐性成本，当心"开源免费"的幻觉
4. **混合策略在成本上更优**：高难度任务用旗舰版、常规任务用性价比版、批处理用 batch API——综合成本更低
5. **定价不是静态的**：2026 年大模型价格仍在快速下降，做长期预算要预留降价空间，不要按当前价格算死
6. **先跑通再优化**：MVP 阶段不需要过度优化成本，验证价值后再做精细化成本控制
