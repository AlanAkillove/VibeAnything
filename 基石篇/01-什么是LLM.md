# 01-什么是 LLM？

> 读完本文你将了解：LLM 是什么、它和普通程序有什么区别、它为什么"看起来"像在思考、以及它的能力边界在哪

---

## 你可能遇到过的问题

你在和 ChatGPT 聊天，它帮你写了一封邮件、解释了一个概念、甚至帮你修改了代码。你觉得它很聪明。

但你可能也遇到过这些情况：

- **同一个问题问两次，给的答案不一样**——而且有时候差别很大
- **它非常自信地给出一个明显错误的答案**——比如让你用不存在的 API
- **你发现它"不懂"简单的数学计算**——3 位数加法都能算错
- **它突然忘记了你前面说过的话**——明明刚才已经确认的细节，现在又不认了

这些现象不是 bug，也不是"AI 出错了"。它们恰恰是理解 LLM 是什么的最好入口。

---

## 常见的误解

在开始之前，先澄清几个大一/大二同学最常见的理解偏差：

| 你以为的 | 实际的 |
|---------|--------|
| LLM 像一个超级聪明的搜索引擎，能从数据库里找答案 | LLM 不会"查"知识，它是在**预测**接下来最可能出现的词语 |
| LLM 能推理、理解、思考，像人一样 | LLM 没有意识、没有理解、没有逻辑——它只做一件事：**算概率** |
| LLM 的回答要么对要么错 | LLM 的回答只有**概率高低**——没有"正确"这个概念 |
| LLM 知道自己的知识边界 | LLM 不知道它知道什么，也不知道它不知道什么 |

理解了这些，你就已经比 90% 的使用者更了解 LLM 了。

---

## 遇到这些问题怎么办？

在了解原理之前，先给你几个马上能用的操作方案：

| 你遇到的问题 | 现在可以怎么做 |
|-------------|---------------|
| 同一个问题两次回答不一样 | 如果是 API，把 temperature 参数设为 0（值越低输出越稳定）；如果是网页版，把有用的回答复制出来再重新生成 |
| AI 自信地给出错误答案 | 在 prompt 里加一句"请一步一步思考"，或者要求它给出信息来源。关键事实要自己去验证 |
| 算不对数学 | 不要让它心算。让它写 Python 代码来算——它能写代码，但不能做算术 |
| 忘记前面说过的话 | 在聊天中主动把关键信息总结发回去。如果是复杂任务，一次提问就把全部上下文说完整 |

等你理解了 LLM 的原理，就会明白为什么这些方案有效。

## 到底什么是 LLM？

**LLM = Large Language Model（大语言模型）**

拆开来看：

### Large（大）
"大"体现在两个维度：

1. **参数数量大**——一个 LLM 通常有数十亿到数万亿个参数。参数可以理解为"神经元之间的连接权重"，数量越大，模型能"记住"的模式就越复杂
2. **训练数据量大**——训练数据通常是整个互联网的文本（TB 级别），包括书籍、论文、网页、代码仓库……几乎你能想到的所有公开文本

> 一个对比：GPT-3（2020 年）有 1,750 亿参数，训练数据合计约 660GB（其中仅 Common Crawl 过滤后就有 570GB）。GPT-4 的参数规模至今未公开，但普遍认为在万亿级别。

### Language（语言）
LLM 处理的是**自然语言**——也就是人类日常使用的文字，不是编程语言。它是"语言"模型，不是"知识"模型、也不是"逻辑"模型。

这意味着：
- 它擅长的是**语言的模式**：语法结构、常见表达方式、概念之间的关联方式
- 它不擅长的是**事实的准确性**：事实对它来说只是"这些词经常出现在一起"的模式

### Model（模型）
"模型"这个词很关键。它不是一个数据库，不是一个搜索引擎，也不是一个程序。它是一个**统计模型**——**函数**。

输入一串文本，输出下一段最有可能的文本。

```
输入: "中国的首都是"
输出: "北京" (概率最高) / "上海" (概率较低) / "南京" (概率更低)
```

就这么简单。没有"理解"、没有"思考"、没有"意识"。它只是从海量文本中学到了"中国的首都是"后面最常出现的是"北京"。

## 一句话解释 LLM 怎么工作的

> LLM 做的事情很简单：给你前半句，它猜后半句。

这个"猜"的过程叫 **Next Token Prediction（下一个词预测）**。

- "Token" 是 LLM 处理文本的最小单位（不是字，也不是词，稍后详细讲）
- LLM 每次生成一个 token，然后把这个 token 加到输入里，再预测下一个 token
- 不断重复这个循环，直到生成完整的回答

**类比：自动补全**

你在手机上打字的时候，输入法会预测你下一个要打的词——`"明天我们去"` → 可能补全 `"吃饭"` / `"看电影"` / `"旅游"`。LLM 本质上就是这个机制的"超级升级版"。

## 传统程序 vs LLM

理解 LLM 和传统程序的区别，是正确使用 AI 的前提：

| 维度 | 传统程序 | LLM |
|------|---------|-----|
| **工作方式** | 遵循程序员写的固定规则 | 从数据中学到的统计模式 |
| **输入** | 结构化数据（表单、JSON） | 自然语言（prompt） |
| **输出** | 确定性的、可预测的 | 概率性的、每次可能不同 |
| **正确性** | 要么对要么错 | 没有"对错"，只有"概率高低" |
| **如何纠错** | 改代码、修 bug | 改 prompt、加上下文 |
| **知识来源** | 程序逻辑中硬编码 | 训练数据中学到的模式 |
| **知识更新** | 改代码重新部署 | 需要重新训练（或加 RAG） |

这个区别解释了文章开头那些"奇怪现象"：

- **答案不一样** → 每次生成是概率抽样，不是确定性计算
- **自信地犯错** → LLM 无法区分"高频模式"和"事实"，它会用同样的自信输出两者
- **算不对数学** → 语言模型不是计算器，它没有"计算"的能力
- **忘记上下文** → LLM 的"记忆"取决于上下文窗口（下一篇详细讲）

## LLM 能做什么、不能做什么

### 能用好的地方

- **文本生成**：写邮件、写文章、写文案
- **代码辅助**：写代码、解释代码、做 Code Review
- **翻译**：多语言互译（尤其是中英文）
- **概括与总结**：长文本提炼要点
- **头脑风暴**：提供创意方向、扩展思路
- **学习辅助**：解释概念、提供例子（就像你现在看的这篇）

### 容易翻车的地方

- **事实性问答**：会编造不存在的引用、数据、事件（这叫 hallucination，后面有专门文章讲）
- **精确计算**：不要让它做算术，用计算器或代码
- **逻辑推理**：复杂逻辑链条容易断
- **时效性**：训练数据有截止日期，它不知道最新的信息（除非联网搜索）
- **机密信息**：不要把密码、API key、隐私数据发给 LLM

## 常见的 LLM 有哪些？

你可能听说过这些名字，它们都是 LLM。在了解有哪些模型之前，先搞清楚一个重要的概念区分。

### 开源模型 vs 闭源模型

在 LLM 领域，"开源"这个词和传统软件领域有重要区别：

| 维度 | 传统开源软件 | LLM "开源" |
|------|------------|-----------|
| **你拿到的是什么** | 源代码（可以编译运行） | 模型权重（训练好的参数文件，不是代码） |
| **能做什么** | 修改代码、重新编译、再发布 | 下载权重、本地运行、微调（部分可以） |
| **不能做什么** | — | 看不到训练数据、看不到训练代码、无法复现训练过程 |
| **许可限制** | 通常宽松（MIT、Apache 2.0） | 常有附加限制（月活限制、商业用途限制） |

**关键区别：** 所谓"开源大模型"绝大多数只是 **open weights（开放权重）**，不是真正意义上的开源。你没拿到训练数据、数据清洗代码、训练代码——这些都是闭源的。Meta 的 Llama 系列是最典型的"开放权重"模型：你可以下载权重自己跑，但训练过程完全不透明。

真正在传统开源意义上做 LLM 的项目极少，目前只有 Pythia、OLMo 等少数几个。

## 你现在应该做什么？

读完这篇文章，你可以在 **ChatGPT / DeepSeek / Kimi** 等任何一个聊天界面中做一个小实验：

1. 问它同一个问题三次（比如"Python 的 list 和 tuple 有什么区别"）
2. 观察三次回答的差异——哪些地方一致，哪些地方不同
3. 问一个明显的事实性问题（比如"2022 年世界杯冠军是谁"）
4. 再问一个它不可能知道的问题（比如"上周末发生了什么新闻"）
5. 观察它怎么"编造"答案的

做完这个实验，你会对"模型"而不是"搜索引擎"有更直观的理解。

**要点总结**
- LLM 是**大语言模型**——从海量文本中学到语言模式的统计模型
- 它的核心机制是 **Next Token Prediction**：猜下一个词
- 它没有"理解"和"思考"，只是在做概率预测
- 它与传统程序有本质区别：概率性 vs 确定性，这是很多奇怪现象的根源
- 知道它能做什么、不能做什么，是用好它的第一步
