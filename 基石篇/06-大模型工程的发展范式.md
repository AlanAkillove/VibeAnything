# 06-大模型工程的发展范式

> 读完本文你将了解：AI 工程化经历了哪几个阶段的发展、每个阶段解决了什么问题、你现在处于哪个阶段、以及接下来该学什么

---

## 你可能遇到过的问题

如果你按照这个教程的顺序读下来，已经了解了 LLM 的原理、Token、提示词、模型生态、能力边界。但你可能会觉得：

- **这些知识之间是什么关系？**——提示词工程和 Agent 有什么区别？RAG 又属于哪个部分？
- **我该先学什么、后学什么？**——那么多概念（RAG、Function Calling、Memory、Agent），有没有一个先后顺序？
- **这个领域变化这么快，我今天学的东西明天还适用吗？**——学"怎么用这个模型"还是学"怎么思考"？

这些问题指向同一个答案：**理解这个领域的工程范式演进**。

---

## 五个阶段的演进

AI 工程化的发展可以概括为五个阶段。**它们不是严格的时间线，而是"关注点"的转移**——后一个阶段不会取代前一个，而是在前一个的基础上叠加。

```
阶段1           阶段2           阶段3           阶段4           阶段5
Prompt       →  Context    →  Agent      →  Harness    →  Loop
Engineering    Engineering    Engineering    Engineering    Engineering

关注"怎么说"     关注"给什么"     关注"做什么"     关注"怎么稳"     关注"怎么好"
```

## 阶段 1：Prompt Engineering（提示词工程）

### 核心问题

**"怎么让 LLM 输出我想要的？"**

这是最基础也是最直接的阶段。LLM 就像一个什么都知道但需要精确指令的新员工。Prompt Engineering 就是学习：怎么跟它说话，它才听得懂。

### 关键能力

- 写清晰的指令
- 角色设定（Role Prompting）
- 示例驱动（Few-shot）
- 思维链（Chain-of-Thought）
- 迭代优化

### 在哪学

已经在 **基石篇/03-提示词工程入门** 中讲过了。

### 局限

Prompt Engineering 能解决"对话"的问题，但解决不了"LLM 不知道你需要的信息"的问题。这就进入了第二阶段。

---

## 阶段 2：Context Engineering（上下文工程）

### 核心问题

**"怎么给 LLM 提供它需要的背景信息？"**

LLM 的训练数据有截止日期，而且它不可能记住你项目的所有细节。Context Engineering 关注的是：**怎么把外部知识有效地送入 LLM 的上下文窗口**。

### 关键技术

- **RAG（Retrieval-Augmented Generation）**：把文档切碎存到向量数据库，检索相关内容后塞进 prompt
- **Memory Bank**：为项目建立持久上下文（设计文档、技术栈、进度追踪），让 AI 在跨 session 时也能记住项目状态
- **System Prompt**：在 API 调用中设定 system message，提供固定的行为指引
- **知识库**：把 FAQ、产品文档、代码库变成 AI 可检索的知识源

### 在哪学

进阶篇/02-RAG与知识库、工具篇/05-Memory Bank。

### 局限

Context Engineering 让 LLM "知道"了更多信息，但它仍然是被动的——你需要告诉它每一步做什么。如果任务需要多步推理、使用工具、自主决策，就需要进入第三阶段。

---

## 阶段 3：Agent Engineering（智能体工程）

### 核心问题

**"怎么让 LLM 自主完成任务，而不是每一步都要我指挥？"**

Agent 的核心是：LLM 作为"大脑"，能理解任务、拆解步骤、调用工具、根据反馈调整。它不再只是一个聊天机器人，而是一个**能行动的实体**。

### 关键技术

- **Function Calling / Tool Use**：LLM 可以调用外部函数（搜索、计算器、数据库查询）
- **任务分解**：把复杂任务拆成子任务，逐步执行
- **自我纠错**：Agent 执行过程中遇到错误，能自己调整策略
- **多 Agent 协作**：不同 Agent 负责不同角色（规划者、执行者、审查者）

### 在哪学

进阶篇/01-Function Calling、进阶篇/03-Agent架构入门。

### 局限

Agent 让 LLM "能做更多事"，但 Agent 系统很脆弱——API 超时、错误响应、无限循环等问题频繁出现。需要第四阶段来解决。

---

## 阶段 4：Harness Engineering（基础设施工程）

### 核心问题

**"怎么让基于 LLM 的系统可靠运行？"**

Harness Engineering 关注的是 LLM 应用的生产化——把"在聊天界面里能用"变成"在生产环境里可靠"。这实际上是把传统软件工程的成熟实践应用到 AI 系统上。

### 关键技术

- **可观测性**：监控 LLM 调用（延迟、Token 消耗、错误率）
- **重试与降级**：API 调用失败时的重试策略、模型不可用时的降级方案
- **安全与合规**：Prompt 注入防护、数据隐私、输出过滤
- **版本管理**：Prompt 版本控制、模型版本管理、A/B 测试
- **成本控制**：Token 预算管理、缓存策略、模型路由

### 在哪学

工作流篇/03-成本意识与Token管理、工作流篇/04-验证闭环。

### 局限

Harness Engineering 让系统 "稳定"了，但系统本身不会变好——它不会从错误中学习，不会自动优化。这需要第五阶段。

---

## 阶段 5：Loop Engineering（闭环工程）

### 核心问题

**"怎么让 AI 系统在运行中持续改进？"**

Loop Engineering 关注的是反馈闭环：AI 系统的输出被评估，评估结果被用来改进系统。这是最前沿的领域，也是区分"玩具"和"产品"的关键。

### 关键技术

- **Evaluation（评估）**：建立自动化的评估 pipeline，衡量输出质量
- **反馈收集**：用户反馈（点赞/点踩）、隐式反馈（用户是否采用了 AI 的建议）
- **数据飞轮**：好的数据 → 更好的模型 → 更好的用户体验 → 更多的数据
- **主动学习**：模型对自己不确定的样本主动请求人工标注

### 在哪学

科研篇/06-实验设计与评估方法、AI协作心法篇/06-会话管理。

## 这些范式对初学者意味着什么？

### 你现在在哪里？

作为大一/大二的学生，你现在最应该关注的是前两个阶段：

| 优先级 | 阶段 | 理由 |
|--------|------|------|
| **P0** | Prompt Engineering | 这是基础中的基础——可以和任何模型交互 |
| **P0** | Context Engineering | 做项目时立刻就能用上（RAG、Memory Bank）|
| **P1** | Agent Engineering | 有趣但需要一定经验，建议先打好前两个的基础 |
| **P2** | Harness Engineering | 做真实项目时自然需要，不必提前学 |
| **P3** | Loop Engineering | 前沿领域，适合有项目经验后再深入 |

### 学范式比学工具更重要

这个领域的变化速度是"以月为单位"。你学的具体工具（某个框架、某个平台的 API）可能在半年后就过时了。但**范式不会**：理解"为什么需要 Context Engineering"比学会某个 RAG 框架重要得多。

这个教程的各个模块，就是围绕这五个范式组织的：

| 范式的哪个阶段 | 教程的对应部分 |
|---------------|--------------|
| Prompt Engineering | 基石篇 03 + AI协作心法篇 |
| Context Engineering | 进阶篇 RAG + 工具篇 Memory Bank |
| Agent Engineering | 进阶篇 Agent + Function Calling |
| Harness Engineering | 工作流篇 + 环境准备篇安全实践 |
| Loop Engineering | 科研篇实验设计 + AI协作心法篇反馈机制 |

## 你现在应该做什么？

1. 回顾这五张图（Prompt → Context → Agent → Harness → Loop），判断你目前在哪一个阶段
2. 如果你的目标是"做一个能用的项目"：重点掌握 1 和 2
3. 如果你的目标是"理解这个领域的未来"：了解 3-5 的方向
4. 不论你的目标是什么：**优先学范式，而不是学工具**

**要点总结**
- AI 工程化经历了五个范式的演进：**Prompt → Context → Agent → Harness → Loop**
- 它们**不是替代关系**，而是逐层叠加——新的范式解决旧范式解决不了的问题
- 对初学者来说，**Prompt Engineering 和 Context Engineering** 是最优先需要掌握的
- **学范式比学工具重要**——工具会变，范式不会
