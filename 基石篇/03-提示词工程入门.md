# 03-提示词工程入门

> 读完本文你将了解：什么是 Prompt、怎么写一个好的 Prompt、常见的 Prompt 技巧、以及如何通过迭代优化得到更好的结果

---

## 你可能遇到过的问题

你开始用 LLM 做一些正经工作了，不只是聊天，而是让它帮你写代码、分析数据、写文档。但你发现：

- **你给的指令不够明确，AI 回答得也很模糊**，“帮我写个函数”→ 它写了一个跟你想要的不一样的函数
- **AI 忽略了你的一些要求**，你说了“用 Python”，它给你写了 JavaScript
- **它总是输出很啰嗦的内容**，你只想要一个简洁的答案，它给了你一篇论文
- **同一个任务，你每次重新写 prompt 效果都不一样**，有时好有时差，完全靠运气

这些问题都不是 AI 的问题，是 **Prompt（提示词）** 的问题。

---

## 常见的误解

| 你以为的 | 实际的 |
|---------|--------|
| Prompt 就是随便问个问题 | Prompt 是需要**精心设计**的输入，就像给一个刚入职的实习生写任务说明 |
| 越长的 Prompt 越好 | 不是越长越好，是**结构清晰 + 信息完整**的 Prompt 更好 |
| Prompt 写一次就够了 | Prompt Engineering 的核心就是**迭代**，写→看结果→改→再看结果 |
| AI 应该能猜到我的意图 | AI 不会读心术，**你要什么、什么格式、什么限制，必须说清楚** |

---

## 遇到这些问题怎么办？

| 你遇到的问题 | 现在可以怎么做 |
|-------------|---------------|
| AI 输出太模糊 | 给更具体的指令：指定角色、输出格式、长度限制 |
| AI 忽略了你的限制 | 限制条件单独成段，放 prompt 靠前或靠后的位置 |
| 输出长度控制不住 | 明确说“用 3 句话回答”或“不超过 200 字” |
| 每次结果差很多 | 降低 temperature，temperature 控制输出的随机性，值越低越稳定可预测）或在 prompt 里加“请给出最稳妥的答案” |
| AI 不理解上下文 | 在提问前加一段背景说明 |

## 什么是 Prompt？

**Prompt（提示词）** 就是你发给 LLM 的输入文本。它可以是：

- 一个简单的问题（"法国的首都是什么？"）
- 一个复杂的任务说明（"请分析这段代码的时间复杂度，并给出优化建议"）
- 一个角色扮演的场景（"你是一名资深 Python 工程师，请 Code Review 以下代码"）

某种意义上，Prompt 就是你**和 AI 之间的编程语言**，你用它来精确表达意图。

---

## 好的 Prompt 长什么样？

一个高质量的 Prompt 通常包含以下要素：

| 要素 | 说明 |
|------|------|
| **角色（Role）** | 让 AI 扮演特定角色 |
| **任务（Task）** | 明确告诉 AI 要做什么 |
| **上下文（Context）** | 提供背景信息 |
| **格式（Format）** | 指定输出的形状 |
| **约束（Constraints）** | 设定边界 |
| **示例（Example）** | 给一个参考样本 |

### 差 Prompt vs 好 Prompt

**差 Prompt：** 帮我写个函数

**好 Prompt：** 你是一名 Python 开发者。请写一个函数，接收一个 URL 列表作为输入，返回其中状态码为 200 的 URL。用 requests 库，加类型注解。输出代码 + 一行使用示例。

## 常用的 Prompt 技巧

### 1. 角色设定（Role Prompting）

让 AI 扮演特定角色，显著提升输出质量。

> 你是一名资深数据科学家。请用最简单的话解释什么是 P 值。

> 你是一名苛刻的 Code Reviewer。请找出以下代码中的所有潜在问题。

### 2. 示例驱动（Few-shot Prompting）

给 AI 提供 1-3 个输入/输出示例，让它学你的模式。

> 将以下中文翻译成地道的英文：
>
> 示例 1：中文：今天天气真好 → 英文：What a beautiful day
>
> 示例 2：中文：这件事让我很为难 → 英文：This puts me in a tough spot
>
> 现在翻译：中文：这个方案虽然可行，但是成本太高了 → 英文：

### 3. 思维链（Chain-of-Thought）

让 AI 一步一步推理，而不是直接给答案。

> 不加思维链：Q: 一公斤羽毛和一公斤铁，哪个更重？A: 铁更重（错误）
>
> 加思维链：Q: 一公斤羽毛和一公斤铁，哪个更重？请一步一步思考。
> A: 都是 1 公斤，所以一样重（正确）

### 4. 步骤分解（Step-by-Step Instructions）

把复杂任务拆成小步骤。

> 1. 先阅读以下代码，理解功能
> 2. 找出潜在的 bug
> 3. 对每个 bug 给出修复方案
> 4. 输出修复后的完整代码

### 5. 迭代优化（Iterative Refinement）

很少有一次写对的 Prompt。好的 Prompt 是**反复修改**出来的。

> 第 1 版 → 结果太啰嗦
> 第 2 版 → 加上"用 3 句话概括"
> 第 3 版 → 加上"用表格输出"
> 第 4 版 → 加上"忽略不重要的细节"
> 第 5 版 → 满意

---

## 一个完整例子

**场景：** 让 AI 写一个 Python 脚本批量重命名文件。

**第 1 版：** "帮我写个批量重命名脚本"
→ 结果太模糊

**第 2 版（加角色和需求）：** "你是一名 Python 开发者。请写一个脚本，批量重命名指定文件夹下的所有 .jpg 文件，编号为 photo_001.jpg。用 pathlib，支持递归，加错误处理"
→ 结果好多了，但还想加自定义前缀和干跑模式

**第 3 版（加约束）：** 加上 dry_run 参数、可指定前缀、用 argparse
→ 最终结果就是你想要的脚本

这个迭代过程就是 **Prompt Engineering 的核心**。

## 你现在应该做什么？

1. 打开任意 LLM 聊天界面
2. 选一个最近做过但不满意的任务
3. 按要素表重新写 Prompt
4. 对比两次结果差异
5. 重复直到满意

**要点总结**
- **Prompt** 是你发给 LLM 的指令，决定了输出质量
- 好的 Prompt 包含：**角色 + 任务 + 上下文 + 格式 + 约束 + 示例**

- **迭代是核心**，没有一次写对的 Prompt，只有不断改好的
