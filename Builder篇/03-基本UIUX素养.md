# 03-基本 UI/UX 素养：不用做设计师，但要有基本 taste

> 读完本文你将了解：为什么非设计师也需要 UI/UX 素养、布局和配色的基本原则、好的交互长什么样、Figma 的基础操作（以及 AI 时代的替代方案）、以及怎么在不需要设计背景的前提下做出「看着不廉价」的产品
> 注：本文的目标不是教你做设计师，而是让你在 Solo 开发时**能做出「不失手」的界面**。你不需要学色彩理论，但要学会「怎么抄得好」

---

## 你可能遇到过的问题

你不是设计师，但你一个独立开发者，需要为自己的 AI 产品做界面：

- **做出来的界面「一看就是程序员写的」**——功能没问题，但就是感觉廉价、粗糙、没有质感
- **不知道颜色怎么配**——要么全白全灰，要么红配绿赛狗屁，要么沉迷于蓝色渐变怎么调都调不对
- **模仿了一个界面但抄得很丑**——明明照着大厂的 UI 元素搬过来，组合在一起就说不出的奇怪
- **花了很多时间调样式**——一个按钮的间距调了半小时，总觉得不对但不知道哪里不对
- **AI 生成的界面时好时坏**——有时候生成的很好看，有时候生成的丑得没法看，不知道怎么鉴别
- **不懂设计师的基本 tooling**——听说要用 Figma，但打开之后完全不知道该点哪里

---

## 常见的误解

| 你以为的 | 实际的 |
|---------|--------|
| 好看的界面需要有审美天赋 | 好看的界面**遵循的是规则，不是天赋**。规则可以学，可以套用 |
| 白板设计要从零开始做 | 大多数好看的产品看起来都差不多——**复用好设计模式**比原创更重要 |
| 设计就是「好看」的东西 | UI 是「好看」，UX 是「好用」。**UX 比 UI 重要**——用得顺手 > 看起来漂亮 |
| 独立开发者不需要懂设计 | 你不需要会做设计，但需要**能鉴别好坏**。你用 AI 生成，你得会判断哪个结果好 |
| Figma 是专业设计师才用的 | Figma 基础操作 30 分钟可以上手，而且 AI 时代有更适合非设计师的工具 |

---

## 常见问题快速解决

| 你遇到的问题 | 现在可以怎么做 |
|-------------|---------------|
| 颜色怎么配都丑 | 直接复制大厂的配色方案——用 shadcn/ui 或 Tailwind 的默认调色盘，不要自己选色 |
| 字体用起来很廉价 | 放弃系统默认字体，用 Inter 或 -apple-system 字体栈；字号遵循 4px 增量规则 |
| 间距不对 | 所有间距用 4px 的倍数：4、8、12、16、20、24、32、48、64，不要出现奇数像素 |
| 布局看起来失衡 | 左对齐一切，不要居中布局（除非是 landing page 的 hero 区） |
| 不知道从哪里学 | 打开你的 Dribbble 或 Mobbin，找一个你觉得好看的应用截图，照着抄它的布局和间距 |
| AI 生成的 UI 风格不统一 | 给 AI 提供明确的 style guide，包括颜色、字体、圆角、间距规则，让它遵守 |

## 三分钟设计原则：布局、配色、交互

你不需要学完整个 UI 设计课程。只要掌握以下三个基本原则，你的界面就能从「能看」变成「还不错」。

---

### 1. 布局：学会用网格和对齐

丑界面最普遍的问题：**元素没有对齐**。

#### 永远不要手动拖拽对齐

不管用 AI 生成还是手写 HTML，遵循以下几个规则：

| 规则 | 怎么做 | 效果 |
|------|-------|------|
| **4px 增量法则** | 所有间距、内边距、外边距使用 4 的倍数（4 / 8 / 12 / 16 / 20 / 24 / 32 / 48 / 64） | 视觉一致性显著提升 |
| **保持 8px 间距网格** | 元素之间的间距至少保持 8px，推荐 16-24px | 界面不会显得拥挤 |
| **左对齐 > 居中** | 除了 Hero 标题和少量装饰文本，正文和控件左对齐 | 用户阅读效率更高 |
| **内容宽度限制** | 正文内容不要全屏宽，限制在 600-720px（文本）或 1024-1200px（工具类） | 阅读更舒适，不会出现超长行 |
| **不要用弹窗居中** | 模态对话框可以用居中，但列表、表单、工具界面不要居中 | 专业感提升 |

**AI 提示词**：「用 Tailwind CSS 帮我生成一个工具页面布局，保持所有间距为 4 的倍数，内容区域最大宽度 1024px，左右间距保持一致。」

#### 借鉴参考：找 3 个参照物

```
1. 找一个你认为好看的工具 / 产品（比如 Linear、Notion、Vercel）
2. 截屏它
3. 分析它的布局结构：
   - 左边是什么宽度？
   - 内容区域离两边有多远？
   - 标题和正文之间间距多大？
4. 照猫画虎复用在你的项目里
```

**好设计都是抄出来的**。不要从零开始布局——先抄、再改、最后才谈得上原创。

### 2. 配色：不要自己调色

配色是独立开发者最容易翻车的地方。解决方案很简单：**不要自己配色，用别人配好的**。

#### 推荐的非设计师配色方案

| 方案 | 适合场景 | 使用方法 |
|------|---------|---------|
| **Tailwind CSS 默认调色板** | 绝大多数工具类产品 | 用 `slate` 或 `gray` 作为 neutral，`blue` 或 `indigo` 作为 accent |
| **shadcn/ui 主题** | React 项目 | 复制 `oklch` 色值，支持 light/dark mode |
| **Radix Colors** | 设计系统级 | 最完善的 accessible 色板，但学习成本略高 |
| **Coolors** | 临时配色 | 随机生成+锁色，比自己选色可靠 |

#### 普通模板 → shadcn/ui 主题示例

```css
:root {
  --background: 0 0% 100%;
  --foreground: 222.2 84% 4.9%;
  --primary: 221.2 83.2% 53.3%;
  --primary-foreground: 210 40% 98%;
  --muted: 210 40% 96.1%;
  --card: 0 0% 100%;
  --border: 214.3 31.8% 91.4%;
}
```

**核心规则**：你的产品最多用 3 种颜色。
- **Neutral 色**（`slate` / `gray`）：占 80% 面积，用于背景、文字、边框
- **Accent 色**（`blue` / `indigo` / `green`）：占 15%，用于按钮、链接、active state
- **警示色**（`red` / `amber`）：占 5%，用于错误、警告

> 如果你不知道用什么 accent 色：Web 工具类用蓝色（`blue-600`），内容消费类用 indigo（`indigo-600`），创作类用紫色（`violet-600`）。这是大厂验证过的规律。

**AI 提示词**：「给我一个 Tailwind CSS 工具的配色方案，使用 slate 作为 neutral，blue 作为 accent。给出各个级别的色值（50 到 900）以及 primary / secondary / muted / destructive 的命名。」

### 3. 交互：让用户用起来「不卡脑子」

UI 是外表，UX 是使用体验。好的 UX 有一条最简单的判断标准：**用户不需要思考就知道下一步该点哪里**。

#### 基础交互原则

| 原则 | 说明 | 反例 |
|------|------|------|
| **一个页面只有一个主要操作** | 确定用户进来要做的「那件事」，让它最显眼 | Landing page 有 5 个不同颜色的按钮，用户不知道该点哪个 |
| **反馈要及时** | 用户操作后 100ms 内给出视觉反馈（loading / 颜色变化 / 动画） | 点了按钮什么都没发生，3 秒后突然跳出结果 |
| **不要打断用户** | 减少弹窗、公告、使用引导 | 一打开产品就弹「订阅 Newsletter」，用户直接关闭 |
| **错误信息要具体** | 不告诉用户「出错了」，告诉他「邮箱格式不对，请检查 @ 符号」 | 「Error 500: Something went wrong」 |
| **默认有值** | 不要让用户面对一个空白输入框不知道填什么，给 placeholder 示例 | 输入框没有任何提示文本 |

#### 交互 vs UI：先做 UX，再调 UI

```
优先级：UX 正确 → UI 干净 → UI 漂亮 → UI 有风格
```

先确保流程走得通：用户 3 步内能完成核心任务。再调样式：颜色、间距、字体。最后才考虑风格：要不要加圆角、阴影、渐变。

## Figma 基础：非设计师的 20 分钟上手指南

你不需要成为 Figma 高手，但需要掌握以下最基础的技能：

### 你需要的 4 个操作

| 操作 | 怎么做 | 什么时候需要 |
|------|-------|------------|
| **画 Frame** | `F` 键 → 选设备尺寸（Desktop / iPhone 14 Pro） | 开始一个新的设计稿 |
| **画矩形和文本** | `R` 键画矩形，`T` 键写文本 | 画出界面的基础元素 |
| **对齐** | 选中多个元素 → 点击顶部工具栏的对齐按钮 | 保持布局整洁 |
| **组合和导出** | `Ctrl+G` 分组，`Ctrl+E` 导出为 PNG/SVG | 和 AI 协作或分享设计 |

### AI 时代的 Figma 替代方案

对非设计师来说，现在有更好的选择——不需要打开 Figma 就能完成设计工作：

| 场景 | AI 工具 | 用法 |
|------|---------|------|
| 生成完整页面设计 | Claude / Cursor / v0 | 自然语言描述 → 生成可直接运行的 HTML |
| 生成 UI 组件 | Claude Artifacts | 「生成一个搜索框组件，带图标和 placeholder」 |
| 生成设计规范 | ChatGPT / Claude | 「把我接下来的项目风格定为：简约、圆角 8px、蓝色 accent」 |
| 调整已有设计 | 截图 + AI 指令 | 截图发给 AI → 「改善这个页面的间距和配色」 |

**独立开发者工作流**：

```
Figma 时代的流程：
想法 → Figma 画稿 → 前端写代码 → 后端对接 → 上线

AI 时代的流程：
想法 → AI 生成 UI 代码 → 调整样式 → 对接后端 → 上线
```

Figma 对独立开发者的价值已经从「画稿」变成了「做参考和沟通」。你不需要用它做完整的设计稿，但你可以在 Figma 里拼凑参考图、梳理布局思路、或者简单截图发给 AI。

## 怎么用 AI 帮你做 UI/UX（具体提示词）

你不需要学会所有设计原则，但需要学会把设计原则转化为 AI 能理解的提示词。

### 场景 1：从零生成一个工具的 UI

```
Prompt：「生成一个 AI 播客摘要工具的单页 HTML。使用 Tailwind CSS。
布局：顶部一个简洁的输入框，用户粘贴播客 URL；中间一个"生成摘要"按钮；
下方是结果展示区。使用 slate 配色，蓝色 accent，8px 圆角。
所有间距使用 4 的倍数。不要多余的装饰。针对桌面 Web 端。」
```

### 场景 2：改善已有界面

```
Prompt：「这是一段现有的 HTML/CSS 代码（粘贴代码）。请帮我改善它的 UI：
1. 使用完善的颜色系统和间距规则
2. 让主要操作按钮更突出
3. 改善排版可读性
4. 保持所有交互有正确的反馈状态
不要改变 HTML 结构，只改 CSS。」
```

### 场景 3：生成设计规范

```
Prompt：「为我的 AI 工具项目生成一个 Design System 文档，包含：
1. 颜色系统（neutral + accent 色）
2. 字体（字体家族、行高、字号级别）
3. 间距（4px grid system）
4. 圆角（8px button corner radius）
5. 阴影级别
6. 按钮的各个状态（default / hover / active / disabled）
用 Tailwind CSS class 表示。」
```

## 你现在应该做什么？

1. 打开你最近做的一个项目，检查它的布局间距——是不是所有 margin/padding 都是 4 的倍数
2. 决定一种配色方案（推荐 Tailwind slate + blue），统一全局颜色
3. 确保你的产品只有一个核心操作按钮，让它最显眼
4. 把 UI 代码发给 AI，让它按今天学到的原则做一次改进
5. 下次做新项目前，先收藏 3 个你觉得好看的同类型产品截图，做参考

## 更进一步

- [Refactoring UI](https://www.refactoringui.com/) — 程序员最好的 UI 设计入门书（不是让你学设计，是让你学会「不犯错」）
- [shadcn/ui](https://ui.shadcn.com/) — 基于 Tailwind + Radix 的组件库，设计基调经过验证，直接复用
- [Tailwind CSS 官方文档](https://tailwindcss.com/docs/customizing-colors) — 内置配色方案，比自选色安全 100 倍
- [Dribbble](https://dribbble.com/) — 找灵感和参考的地方，注意不要被太花哨的作品带偏，专注找工具类产品

---

**要点总结**
- 好设计遵循规则，不是天赋。**4px 增量法则、左对齐、内容区域限制**是最基本的三个布局规则
- 不要自己配色，**用现成的配色方案**（Tailwind / shadcn/ui / Radix Colors）
- 三个颜色控制全局：**Neutral（80%）+ Accent（15%）+ 警示（5%）**
- **UX > UI**：先确保流程走得顺，再调整样式。一个页面只有一个主要操作
- Figma 对非设计师来说 20 分钟可以入门，但**AI 时代有更好的替代方案**
- 把你的设计原则转化为 AI 提示词——给 AI 讲清楚规则，它的输出质量会大幅提升
- **好设计都是抄出来的**。先收藏参考、再复用、最后才谈原创
