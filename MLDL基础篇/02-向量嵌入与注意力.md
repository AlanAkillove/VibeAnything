# 02-向量、嵌入与注意力

> 读完本文你将了解：计算机怎么"理解"文字、什么是词向量和嵌入、注意力机制是怎么回事、以及这些概念怎么构成了现代 LLM 的基础

---

## 你可能遇到过的问题

你已经理解了神经网络的基本直觉，但还有一个根本问题没想通：

- **计算机只认识数字，文字是怎么"喂"给神经网络的？**——这中间肯定有个转换过程
- **"嵌入"这个词经常听到**——但到底嵌入到什么里面？
- **你听说 Transformer 的核心是"注意力机制"**——注意力是什么？机器也会"注意"？
- **你知道 Attention Is All You Need 这篇论文很重要**——但读了开头就看不懂了

这些问题的答案，全都在**如何把文字变成数字**这个过程中。

---

## 常见的误解

| 你以为的 | 实际的 |
|---------|--------|
| AI 在处理"文字"，所以它认识字 | AI 从来没见过"文字"——它只处理**数字序列** |
| 一个词对应一个固定的编码 | 一个好的嵌入方案里，**相同含义的词在不同上下文中可以有不同表示** |
| "注意力"就是模型"关注"某部分 | 注意力是一个**数学运算**——计算"哪些部分对当前输出更重要"并加权 |
| Transformer 很难理解 | 核心概念只有一个：**注意力机制**。理解了它，就理解了 Transformer 的一半 |

## 第一步：把文字变成数字

计算机不认识"猫"这个字，它只认识数字。所以我们需要一种方式，把文字映射成数字。

### 最简单的方案：One-hot 编码

假设我们的词汇表只有 5 个词：`[猫, 狗, 鱼, 鸟, 鼠]`

每个词用一个向量表示，某一位为 1，其余为 0：

```text
猫 → [1, 0, 0, 0, 0]
狗 → [0, 1, 0, 0, 0]
鱼 → [0, 0, 1, 0, 0]
```

这个方案有一个大问题：**所有词之间的距离都一样**。"猫"和"狗"的距离 = "猫"和"鼠"的距离。但直觉告诉我们，"猫"和"狗"应该更接近才对（都是宠物），而"鼠"更远。

### 更好的方案：嵌入向量

嵌入（Embedding）的思想是：**为每个词学习一个向量（几十到几千维），让含义相近的词在向量空间中靠在一起。**

```
2D 空间中的词嵌入（简化版）：
            ↑ 温暖
            |
   [太阳]   |
            |
  [火]      |    [狗]
  [热]      |    [猫]
            |
────────────┼────────────→ 可爱
            |   [鱼]
 [冰]       |
 [雪]       |
            |
            ↓ 寒冷
```

在这个空间里，"猫"和"狗"的距离很近，"冰"和"雪"的距离很近，而"猫"和"雪"的距离很远。**这种距离关系就是模型对"词义"的表示。**

这个向量空间就是"嵌入空间"——词被"嵌入"到了一个有含义的数学空间中。

---

## 第二步：从词嵌入到句子的理解

单个词的嵌入只能表示词义。但**一句话的意思不只是词的堆叠**——"猫追狗"和"狗追猫"包含同样的词，意思完全不同。

所以模型需要知道**词与词之间的关系**。这里用的核心机制就是**注意力（Attention）**。

---

## 注意力机制直觉

### 一个生活类比

你在看一张照片。你的眼睛不会同时注意照片上的每一个像素——你会**聚焦**在几个关键区域。

- 先看人脸 → 再看一个人的眼睛 → 再看这个人在指着什么 → 顺着手指看过去

这个"聚焦"的过程，就是注意力。

在 LLM 中，当模型处理某个词时，它也会"注意"到句子中其他相关的词。

### 一个具体例子

处理一句话："它因为没吃饱，所以很生气。"

当模型读到"它"时，它需要知道"它"指的是什么。注意力机制会计算：

```text
"它" 应该注意句子中的哪些词？

"它" → 没有相关信息
"因为" → 连接词
"没" → 否定
"吃饱" → 这个词和"它"的关系最大
"所以" → 连接词
"很" → 程度
"生气" → 和"吃饱"一起解释"它"的状态
```

模型通过**计算词与词之间的关联分数**来确定每个词应该"注意"其他哪些词、注意多少。这就是注意力机制在做的事。

---

## QKV：理解注意力的核心

在 Transformer 的注意力机制里，每个词被赋予了三种角色：

| 角色 | 名字 | 类比 |
|------|------|------|
| **Query（查询）** | 当前词在问："谁和我相关？" | 你在图书馆想找书——"有哪些书是关于神经网络的？" |
| **Key（键）** | 其他词在答："我在这里" | 每本书的分类标签 |
| **Value（值）** | 被关注后"贡献的内容" | 书里真正的内容 |

过程简化版：

```
1. "它" 的 Query 去匹配所有词的 Key
2. 发现"吃饱"的 Key 匹配度最高
3. "吃饱"的 Value 被加权传递给"它"
4. "它" 的表示中融入了"吃饱"的信息
```

这个 QKV 机制让模型能够**动态地确定信息流动的方式**——不像之前的模型（RNN）那样只能按顺序逐个传递信息。

---

## 为什么注意力这么重要

在 Transformer 出现之前（2017 年之前），主流的序列模型是 RNN（循环神经网络）。RNN 的问题是：**只能按顺序逐个处理词，离得远的词之间信息很难传递。**

```
RNN 的处理方式（串行）：
我 → 昨天 → 在 → 公园 → 看到 → 一只 → 很可爱的 → → → → → → 猫
                                                       信息很难传这么远

Transformer 的处理方式（并行 + 注意力）：
我  昨天  在  公园  看到  一只  很可爱的  猫
 ↑    ↑    ↑    ↑    ↑    ↑     ↑      ↑
 └────────────── 注意力 ────────────────┘
                                   所有词之间可以直接通信
```

注意力机制让 LLM 能"看到"整个上下文，而不是只看附近几个词。**这是 LLM 能做长文本理解的关键。**

---

## Self-Attention 和 Masked Attention

| 类型 | 说明 | 用在哪儿 |
|------|------|---------|
| **Self-Attention** | 每个词和句子中所有其他词计算注意力 | 编码器（BERT 等） |
| **Masked Self-Attention** | 每个词只和它前面的词计算注意力 | 解码器（GPT 等） |

LLM 大多使用**解码器架构**（Decoder-only），所以用的是 Masked Self-Attention——预测下一个词时，只能看到已经出现的词。

---

## 从这些概念到 LLM

| 本文的概念 | 在 LLM 中对应 |
|-----------|--------------|
| 词嵌入 | LLM 的第一层把所有 token 转成向量 |
| 注意力机制 | Transformer 的核心计算单元 |
| QKV | 注意力机制的具体实现方式 |
| Masked Self-Attention | GPT 系列模型生成文本的方式 |
| 多层注意力 | 几十层 Transformer 堆叠，每层都做注意力计算 |

**要点总结**
- 计算机不认识文字——**所有文本都要先转成数字序列**（嵌入向量）
- **词嵌入**让含义相近的词在向量空间中靠在一起——这就是"理解"词义的方式
- **注意力机制**让模型在处理一个词时，能动态关注句子中所有其他词
- QKV 是注意力的具体实现：**Query 问"谁和我相关"，Key 回答"我在这里"，Value 贡献"我的内容"**
- Transformer 的核心创新就是注意力——它让模型能**并行处理整个序列**，而不是逐个串行
