# 02-向量、嵌入与注意力：大模型是怎么「理解」文字的

> 读完本文你将了解：计算机怎么把文字变成可计算的数字、词嵌入为什么能表示语义、注意力机制的核心逻辑，以及这些组件如何构成了现代大模型的基础
> 注：本文全程以直觉理解为主，不涉及复杂公式推导，零基础也能看懂核心原理。

## 你可能遇到过的问题

你已经理解了神经网络的基本逻辑，但始终绕不开一个核心疑问：
- **计算机只认识数字，文字、词语到底是怎么喂进神经网络的？** 这中间一定有个转换过程
- 总听「嵌入/Embedding」这个词，到底是什么东西嵌入到哪里了？
- 同一个词在不同句子里意思完全不同，AI能分清楚吗？
- Transformer 的核心「注意力机制」到底是什么？机器也会像人一样「集中注意力」？
- 听说《Attention Is All You Need》是AI领域的神文，但翻开就满页公式，完全无从下手

这些问题的答案，全都围绕「如何把文字变成有语义的数字」这个核心过程展开。

## 常见的误解

| 你以为的 | 实际的 |
|---------|--------|
| AI处理的是文字本身，它认识汉字/单词 | AI从来没见过「文字」，它全程处理的都是**数字向量序列** |
| 一个词对应一个固定编号就足够表示 | 好的表示方案里，**词的含义、上下文关系都会体现在向量中**，远不是简单编号 |
| 「注意力」就是模型像人一样「盯着看」某部分内容 | 注意力本质是**加权求和的数学运算**：计算每个词对当前词的重要程度，按权重融合信息 |
| Transformer结构非常复杂，入门门槛很高 | Transformer的核心只有一个：**自注意力机制**。搞懂它，就理解了大模型的大半 |

## 第一步：把文字变成可计算的数字

计算机天生只认识数字，要让神经网络处理文字，第一步就是把文字映射成可计算的数字形式。

### 最朴素的方案：One-Hot 独热编码
假设我们有一个极小的词汇表：`[猫, 狗, 鱼, 鸟, 鼠]`
给每个词分配一个固定位置，对应位置为1，其余为0，就是独热编码：
```
猫 → [1, 0, 0, 0, 0]
狗 → [0, 1, 0, 0, 0]
鱼 → [0, 0, 1, 0, 0]
```

但这个方案有两个致命缺陷：
1. **维度爆炸**：真实场景的词汇表有几万甚至几十万个词，每个词就是几十万维的向量，绝大多数都是0，计算效率极低
2. **完全无语义**：所有词之间的距离都相等。「猫」和「狗」的距离 = 「猫」和「鱼」的距离，但常识里猫和狗都是宠物，语义显然更近。

### 真正的解决方案：词嵌入（Word Embedding）
嵌入的核心思想是：**让模型从海量文本里自动学习每个词的向量表示，让含义相近的词在高维向量空间里自然靠在一起**。

我们可以把高维向量压缩到二维平面，直观感受语义空间：
```
            ↑  温暖
            |
   [太阳]   |
            |
  [火]      |    [狗]
  [热]      |    [猫]
            |
────────────┼────────────→  可爱
            |   [鱼]
 [冰]       |
 [雪]       |
            |
            ↓  寒冷
```

在这个空间里：
- 「猫」和「狗」距离很近，因为同属宠物范畴
- 「冰」和「雪」距离很近，因为都和寒冷属性相关
- 「猫」和「雪」距离很远，语义几乎没有关联

**向量之间的距离，就代表了语义的相似度**。这个把词映射到有语义的高维数学空间的过程，就叫「嵌入」——词被嵌入到了一个承载语义信息的向量空间里。

#### 最经典的直觉：向量也能做语义运算
嵌入最神奇的特性，是向量加减法可以对应语义关系的变化：
> 国王 - 男人 + 女人 ≈ 女王

没有人教模型「性别」「身份」这些抽象概念，但它在海量文本的训练中，自动把这些属性学到了向量的方向和距离里。

> 补充说明：嵌入向量不是人为设计的规则，完全是模型在大规模训练中自动学习出来的。向量维度通常从几十到几千不等，维度越高，能容纳的语义细节越丰富。

## 第二步：注意力解决「词与词的关系」

单个词的嵌入只能表示孤立的词义，但一句话的意思远不是词的简单堆叠。
比如「猫追狗」和「狗追猫」，用到的词完全一样，但意思截然相反。再看这个更典型的例句：
> The animal didn't cross the street because it was too tired.

句中的「it」到底指代「animal」还是「street」？人一眼就能判断，但模型需要明确捕捉词与词之间的指代、修饰、因果关系。

**注意力机制（Attention），就是用来解决这个问题的：让模型在处理每个词的时候，动态地关注句子里其他相关的词，把它们的信息加权融合进来。**

### 注意力的直觉理解
你看一张合照的时候，不会均匀盯着每一个像素。你会先聚焦人脸，再看人物的动作，最后扫过背景——你的视线会根据当前的关注点，动态分配精力。

模型处理文字也是同理：处理「it」这个词的时候，不需要平均关注所有词，只需要重点关注「animal」「tired」这些强相关的词，忽略无关的连接词即可。

### QKV：注意力的三套核心角色
Transformer里的自注意力（Self-Attention），是通过给每个词分配三个不同功能的向量实现的，可以用图书馆找书的场景类比理解：

| 缩写 | 全称 | 核心作用 | 通俗类比 |
|------|------|----------|----------|
| **Q** | Query 查询 | 当前词发出提问：「句子里谁和我最相关？」 | 你想找书，说出搜索关键词 |
| **K** | Key 键 | 每个词亮出自己的「身份标签」，等待匹配 | 每本书的书名、分类标签 |
| **V** | Value 值 | 每个词实际承载的语义内容 | 书里的具体正文内容 |

整个计算过程可以简化成三步：
1. **计算匹配度**：用当前词的 Q 与所有词的 K 计算相似度，得到每个词的关联分数
2. **归一化权重**：把分数转换成 0~1 之间的权重，所有权重加和为1
3. **加权求和**：用权重乘以每个词的 V 再相加，就得到了当前词融合上下文后的新表示

回到之前的例句：处理「it」的时候，计算后「animal」和「tired」的权重会显著高于其他词，它们的语义信息就会融合进「it」的向量里，模型就能准确判断「it」指代的是动物，而非街道。

### 多头注意力：从多个角度理解语义
真实的 Transformer 不会只做一次注意力计算，而是同时并行多组（常见8、16、32组），这就是**多头注意力（Multi-Head Attention）**。

不同的注意力头会自发关注不同类型的语义关系：
- 有的头关注指代关系（代词对应哪个名词）
- 有的头关注语法修饰（形容词修饰哪个名词）
- 有的头关注因果逻辑（关联词连接的前后文）

多组注意力并行计算，最后拼接结果，模型就能从多个维度捕捉句子的复杂关系。

### 不能漏掉的细节：位置编码
自注意力本身是「无序」的——它只计算词和词的关联度，不关心谁在前谁在后。但文字的顺序直接决定了语义，「我打他」和「他打我」用词完全相同，含义彻底相反。

所以 Transformer 专门给每个位置的词叠加了一段**位置编码（Positional Encoding）**向量，和词嵌入相加后再送入网络。位置编码有固定的生成规律，能让模型精准感知每个词的绝对位置、以及词与词之间的相对距离。

## 为什么注意力是革命性的：和前代模型的对比

在 2017 年 Transformer 论文发表之前，处理文本的主流模型是 RNN（循环神经网络）。

### RNN：串行处理，长距离信息易丢失
RNN 必须严格按顺序，一个词接一个词往后处理，像串珠子一样：
```
我 → 昨天 → 在 → 公园 → 看到 → 一只 → 很可爱的 → 猫
```
- 信息只能一步步向后传递，离得远的词之间信息很难传递到位，长文本理解能力弱
- 必须等前一个词处理完才能处理下一个，无法并行计算，训练速度非常慢

### Transformer + 注意力：全并行，全局直接交互
有了自注意力，句子里所有词可以同时处理，任意两个词之间都能直接建立关联：
```
我  昨天  在  公园  看到  一只  很可爱的  猫
 ↑    ↑    ↑    ↑    ↑    ↑     ↑      ↑
 └────────────── 自注意力 ────────────────┘
```
- 任意距离的词都能直接传递信息，长文本理解能力大幅提升
- 所有词可以并行计算，训练效率指数级提高，这也是大模型能越做越大、越训越快的核心基础

## 两种自注意力：双向理解 vs 单向生成

根据能否看到完整的句子，自注意力分为两种，对应不同的模型架构：

| 类型 | 核心规则 | 典型应用场景 |
|------|----------|--------------|
| **自注意力（Self-Attention）** | 每个词可以看到句子里全部其他词，双向交互 | 编码器架构，比如 BERT，适合文本理解、分类类任务 |
| **掩码自注意力（Masked Self-Attention）** | 每个词只能看到它**前面**的词，后面的词会被「掩码」遮住 | 解码器架构，也就是 GPT 系列大模型采用的方案，适合文本生成 |

我们日常使用的 ChatGPT、文心一言等生成式大模型，都属于解码器-only 架构，用的就是掩码自注意力——预测下一个词的时候，只能依赖已经生成的前文，不能「偷看」后面的答案，完全符合逐字生成文本的逻辑。

## 这些概念和大模型（LLM）的对应关系

你现在理解的这些基础组件，拼起来就是大模型的完整骨架：

| 基础概念 | 在大模型中的对应 |
|-----------|--------------|
| 词嵌入 + 位置编码 | 模型的最底层输入层，把文字 token 转换成向量，并补充位置信息 |
| 多头自注意力 | 每个 Transformer 层的核心计算单元，负责融合上下文语义 |
| 掩码自注意力 | 生成式大模型的标准配置，保证逐词生成的逻辑合理性 |
| 多层 Transformer 堆叠 | 主流大模型有几十到上百层 Transformer 层，逐层提炼抽象语义 |

## 要点总结
1. 计算机不认识文字，**所有文本都要先转换成向量数字**才能送入神经网络，这个转换过程就是嵌入
2. **词嵌入**的核心是语义空间：含义相近的词在向量空间中距离更近，甚至可以通过向量加减法实现语义运算
3. **自注意力机制**解决了「词与词的关系」问题：通过 QKV 计算每个词的重要程度，加权融合上下文信息
4. 多头注意力让模型从多个维度理解语义；位置编码补充了词的顺序信息，弥补自注意力无序的缺陷
5. 对比前代 RNN，注意力机制实现了**全局并行计算**，既提升了长文本理解能力，也大幅提高了训练效率，是大模型规模化的基础
6. 生成式大模型采用**掩码自注意力**：预测下一个词时只能看到前文，保证生成逻辑的合理性
