# 03-从预训练到对话：大模型是怎么练成聊天助手的

> 读完本文你将了解：大模型从「读遍互联网文本」到「能和你流畅对话」经历了哪几个阶段、预训练和微调的核心区别、SFT 和 RLHF 分别在做什么，以及为什么训练大模型需要消耗海量算力
> 注：本文延续直觉优先的风格，不涉及算法细节，搞懂流程和核心逻辑即可建立完整认知。

## 你可能遇到过的问题

你已经理解了神经网络、嵌入和注意力的基础原理，但始终有一个核心疑问没打通：
- 模型明明只是「看了很多书」，怎么就变成能回答问题、陪人聊天的助手了？看书和聊天明明是两件完全不同的事
- 总说大模型「训练成本极高」，训练到底在算什么？为什么需要几千张 GPU 跑几个月？
- SFT、微调、RLHF 这些名词满天飞，它们之间是什么关系？分别对应训练的哪个阶段？
- 为什么同样是 Transformer 架构，有的模型只能续写文本，有的就能做对话助手？

这些问题的答案，串起来就是现代大语言模型的完整训练路径：**预训练 → 监督微调 → 人类反馈对齐**。三步走完，一个通用的文本模型就变成了符合人类使用习惯的对话产品。

---

## 常见的误解

| 你以为的 | 实际的 |
|---------|--------|
| 训练大模型就是给它「读」海量的书和网页 | 不是「读书理解」，核心任务是**预测下一个词**：给一段文本的前半部分，让模型猜下一个最可能出现的词 |
| ChatGPT 是人工一条条教出来的 | 模型 99% 以上的知识都是在预训练阶段「自学」的，人工只参与少量的格式引导和偏好对齐 |
| 预训练完的模型就能直接聊天 | 预训练模型本质是「文本补全器」，只会顺着开头续写。**必须经过微调，才能学会「用户提问我回答」的对话模式** |
| 训练一个大模型几天就能完成 | GPT-3 级别模型的预训练需要数月，旗舰级大模型的全流程训练成本可达数千万到上亿美元 |

## 训练全流程总览：三步练成对话助手

```
预训练（Pre-training） ──→ 监督微调（SFT）──→ 人类反馈对齐（RLHF）
     （学语言规律）            （学对话格式）          （学人类偏好）
```

| 阶段 | 核心目标 | 核心原料 | 产出的模型能力 |
|------|----------|----------|----------------|
| 1. 预训练 | 学会语言的语法、语义、事实知识和通用规律 | 全网海量文本（书籍、网页、代码、论文等） | 基础语言模型：给任意开头，能流畅续写文本 |
| 2. 监督微调（SFT） | 学会「提问-回答」的对话格式，遵循指令 | 人工标注的高质量问答对、指令数据 | 指令跟随模型：能听懂用户指令，直接给出答案 |
| 3. RLHF 对齐 | 让输出更符合人类偏好：更有用、更安全、更通顺 | 人类对模型回答的排序评价 | 对话助手：回答自然、贴合需求、符合人类使用习惯 |

## 第一步：预训练（Pre-training）——打下所有能力的基础

预训练是整个流程中成本最高、数据量最大、耗时最久的一步，也是模型所有能力的根基。

### 核心任务：预测下一个词
预训练的任务朴素到让人意外：给模型看一段文本的前半部分，让它预测下一个词是什么。
```
输入：「今天天气真」
模型输出各词的概率：
好：87%
棒：6%
不错：4%
差：2%
……
```
这个任务正好和我们上一篇讲的**掩码自注意力**完美匹配：预测当前词时，模型只能看到它前面的所有词，不能偷看后面的内容，保证了生成逻辑的自洽性。

你在 ChatGPT 里看到的逐字输出，本质就是这个过程的循环：每生成一个新词，就把它加回输入序列里，继续预测下一个，直到生成结束标记。这种「用自己的输出当输入继续生成」的模式，叫做**自回归（Auto-regressive）**。

### 训练数据的规模
预训练的数据量是真正的「海量」。以 GPT-3 为例，训练数据包含约 5000 亿个 Token，大概相当于 100 倍的英文维基百科总量。
数据来源覆盖：
- 全网网页抓取（Common Crawl）
- 公开书籍、学术论文、代码仓库
- 百科、论坛、新闻、公开文档

### 训练为什么这么费算力
训练的过程就是不断重复「预测 → 和正确答案对比 → 计算误差 → 反向调整参数」的循环：
1. 输入一段文本，模型预测下一个词
2. 和真实的下一个词对比，算出错了多少（损失函数）
3. 通过反向传播算法，从后往前调整模型里所有的权重参数
4. 让下一次预测的结果更接近正确答案

因为模型参数有上千亿甚至万亿，训练数据有几千亿 Token，每一步都要做超大规模的矩阵运算，所以需要上千张 GPU 连续跑几个月才能完成。这也是大模型训练成本极高的核心原因。

> 通俗类比：预训练就像一个人读了几百万本书，没有老师教他答题，只是在海量阅读中慢慢摸透了语言的规律、世界的常识、逻辑的模式。他能流畅写文章，但还不知道怎么回答别人的提问。

## 第二步：监督微调（SFT）——学会「回答问题」的格式

### 为什么必须有这一步
预训练模型只会一件事：顺着输入续写文本。它完全没有「用户提问 → 我给出答案」的意识。

举个直观的例子：
给预训练模型输入「法国的首都是什么？」，它大概率会续写：
> 法国的首都是什么？这是初中地理的经典考点，常出现在欧洲地理章节的练习题中，主要考察学生对国家首都的记忆……

它会顺着「问题」这个文本继续往下写文章，而不是直接给出答案。因为它的训练数据里，问题后面跟着的往往是更多的题干、解析，而不是直接的答案。

### SFT 到底做了什么
SFT 的全称是 **Supervised Fine-Tuning（监督微调）**，核心就是用人工整理好的「指令-回答」配对数据，继续训练模型，让它学会对话的格式。

训练数据长这样：
```
输入（用户指令）：「法国的首都是什么？」
期望输出（标准答案）：「法国的首都是巴黎。」
```

模型看成千上万个这样的问答对之后，就慢慢摸到了规律：哦，原来这种带问号的、像提问的输入，我不能续写文章，要直接给出对应的答案。

这一步需要的 data 量比预训练小得多，通常只需要几万到几十万条高质量的标注数据，就能让模型从「文本补全器」变成「指令跟随者」。

> 通俗类比：就像一个饱读诗书的人，之前只会自己写文章，现在给他看几千道「题目+参考答案」，他很快就学会了考试答题的格式——看到题目，就直接写答案，而不是续写题目。

## 第三步：RLHF 对齐——让回答更符合人类偏好

### 为什么 SFT 之后还不够
经过 SFT 的模型已经能回答问题了，但回答质量往往差强人意：
- 有时候啰嗦半天说不到重点，有时候又太简短
- 会一本正经地输出错误信息（幻觉）
- 语气生硬，或者不符合人类的交流习惯
- 可能会输出不安全、不合规的内容

RLHF 要解决的就是这个问题：**让模型的输出不仅格式正确，还要更贴合人类的偏好——更有用、更安全、更通顺。**
RLHF 的全称是 **Reinforcement Learning from Human Feedback（基于人类反馈的强化学习）**。

### 三步拆解 RLHF
1. **人类标注排序**：同一个问题，让模型生成多个不同的回答。标注员按照「好坏」给这些回答排序，比如 B 回答比 A 好，A 比 C 好。
2. **训练奖励模型**：用人类的排序数据，训练一个专门的「奖励模型」。这个模型的作用是：给任意一段模型回答，自动打出一个「人类偏好分」，不用人再手动评价。
   > 为什么不直接让人打分？因为人工评价速度太慢了，根本跟不上训练节奏。先训练奖励模型，就能实现自动化批量打分，效率提升无数倍。
3. **用奖励优化主模型**：把奖励模型当作「裁判」，主模型每生成一个回答，裁判就给它打分。主模型的优化目标就是：想办法生成能拿到更高分的回答。

整个过程下来，模型就会慢慢朝着人类喜欢的方向调整：回答更有条理、信息更准确、语气更自然、也更安全合规。

> 通俗类比：就像学生写完作文，老师每次都给他打分、说哪里写得好哪里不好。学生写得多了，慢慢就摸透了老师的偏好，写出来的作文越来越符合高分标准。

## 核心概念速查表

| 术语 | 一句话解释 | 生活化类比 |
|------|------------|------------|
| **预训练** | 在海量文本上做下一词预测，学会通用语言规律 | 读万卷书，打下知识和语言基础 |
| **监督微调（SFT）** | 用问答对训练，让模型学会遵循指令、回答问题 | 刷真题，学会答题的格式和思路 |
| **RLHF** | 根据人类反馈优化输出风格，对齐人类偏好 | 根据老师的批改调整作文，越写越符合高分标准 |
| **推理** | 用训练好的模型处理用户输入、生成回答 | 上考场答题，用已有的知识输出结果 |
| **自回归** | 逐词生成，每生成一个词就放回输入继续预测下一个 | 边想边说，说出来的话也会影响后面的表达 |

## 知识闭环：回看之前的疑问

学到这里，你应该能解答基石篇留下的这些问题了：

| 之前的疑问 | 现在的答案 |
|-----------|------------|
| Token 到底是什么？ | 文本的最小处理单位，先被转成嵌入向量，再送入 Transformer 网络计算 |
| 为什么同一个词在不同句子里意思不一样？ | 自注意力机制会让每个词的表示融合上下文信息，动态调整语义 |
| 为什么大模型会「记不住」前面说的话？ | 受上下文窗口限制，注意力只能处理窗口内的 Token，超出范围的信息模型看不到 |
| 提示词工程为什么有用？ | 你写的提示词会作为上下文，引导注意力聚焦在关键信息上，影响模型的输出方向 |
| 为什么大模型会胡说八道（幻觉）？ | 模型学到的是语言的统计规律，不是事实本身。它会生成「最可能出现的文本」，而不一定是「最正确的文本」 |

## 要点总结
1. 现代大模型的训练分为三步：**预训练学语言 → SFT 学对话 → RLHF 学偏好**，三步走完才能成为好用的对话助手
2. **预训练**是成本最高的一步，核心任务就是「预测下一个词」，靠几千亿 Token 的海量数据打下所有能力的基础
3. **SFT 监督微调**用少量人工问答对，把「文本补全器」调教成「指令跟随者」，让模型学会回答问题的格式
4. **RLHF** 通过人类排序数据训练奖励模型，再用奖励优化主模型，让输出更符合人类的使用偏好
5. 我们日常和 AI 聊天属于**推理阶段**，模型参数是固定的，不会因为对话而「学习新知识」，只是用已有参数逐词生成回答
