# 03-从预训练到对话

> 读完本文你将了解：LLM 是怎么从"读了一堆书"变成"能跟你聊天"的、预训练和微调有什么区别、以及 RLHF 是什么意思

---

## 你可能遇到过的问题

你已经知道了神经网络的基本原理，也理解了嵌入和注意力。但还有一个核心问题没解决：

- **一个模型怎么从"看了很多书"变成"能回答我的问题"？**——看书和聊天是完全不同的两件事
- **GPT 的"训练"到底是怎么回事？**——所有人都说它"训练"了，但没人说清楚具体过程
- **为什么需要那么多 GPU 和那么多钱？**——训练一个模型到底在做什么，那么费资源
- **你听说过 RLHF、SFT、Fine-tuning 这些词**——但不知道它们之间是什么关系

这些概念可以串成一条线：**预训练 → 微调 → RLHF**。这是现代 LLM 训练的完整路径。

---

## 常见的误解

| 你以为的 | 实际的 |
|---------|--------|
| 训练 LLM 就是"给它读很多书" | 不是"读"，是**预测下一个词**——每次看一段文本，遮住最后一个词，让它猜 |
| GPT 是被人"教"出来的 | GPT 的大部分知识是**自学**的——它从互联网文本中自己发现了语法、事实、推理模式 |
| 预训练好的模型就能直接聊天 | 预训练模型是一个"文本补全器"——你给开头，它续写。**需要微调才能变成对话助手** |
| 训练一个模型需要几天 | 训练 GPT-4 级别的模型需要**几个月**，成本在数千万到数亿美元 |

---

## 三步走：从互联网文本到你的聊天助手

```
预训练 ──→ 监督微调（SFT）──→ RLHF
 (学语言)      (学对话)        (学偏好)
```

| 步骤 | 做什么 | 需要什么 | 结果 |
|------|--------|---------|------|
| 1. 预训练 | 在互联网文本上预测下一个词 | 海量文本 + 大量 GPU | 基础模型：能续写文本 |
| 2. 监督微调 | 用高质量的对话数据训练 | 人工编写的问答对 | 指令模型：能回答问题 |
| 3. RLHF | 对齐人类的偏好 | 人类对模型输出的评价 | 对话助手：更符合用户期望 |

---

## 第一步：预训练（Pre-training）

### 任务：预测下一个词

预训练的任务极其简单——给你一段文字的前半部分，预测下一个词。

```text
输入："今天天气真"
输出预测：各词的概率
好：87%
棒：8%
不：3%
差：2%
```

模型在看到"今天天气真"之后，根据它在训练数据中学到的模式，认为"好"最可能出现在后面。

### 训练数据有多大

GPT-3 的训练数据包含大约 **5000 亿个 token**——大约相当于整个英文 Wikipedia 的 100 倍。

这些数据来自：
- Common Crawl（互联网抓取）
- 书籍、论文、代码库
- Wikipedia、Reddit、新闻

### 训练过程

模型不会一次性看完所有数据。它一遍又一遍地看，每次看到一段文本，都试图预测下一个词。**预测对了，增强对应的参数链接；预测错了，调整参数让下次更可能对。**

这个过程叫"反向传播"（Backpropagation）——不要求理解细节，只需要知道：它不断重复"预测 → 对错的信号反馈 → 调整参数"这个循环。

**GPT-3 的训练持续了几个月，使用了上千张 GPU。**

---

## 第二步：监督微调（SFT）

### 为什么需要

预训练模型只学会了一件事：**补全文本**。

如果你给它 "法国的首都是"，它可能续写 "法国的首都是巴黎，也是欧洲的文化中心之一。"

但它不知道怎么回答 "法国首都是什么？"——因为它的训练数据里，问题后面跟的不是回答，而是更多的文章内容。

### SFT 做了什么

SFT（Supervised Fine-Tuning）用**人工编写的问答对**来训练模型：

```text
输入："法国的首都是什么？"
期望输出："法国的首都是巴黎。"
```

模型看到成千上万个这样的（问题 → 回答）对后，学会了"别人问我问题，我应该给出答案"这个模式。

这步需要**人工标注**——人写问题和答案。ChatGPT 在 SFT 阶段用了数千条人工标注的问答对。

---

## 第三步：RLHF 对齐

### 为什么还需要第三步

SFT 之后，模型能回答问题了，但回答的质量参差不齐：
- 有时太啰嗦，有时太简短
- 有时给出错误信息
- 有时语气不合适

RLHF（Reinforcement Learning from Human Feedback）要解决的是：**怎么让模型的输出符合人类的偏好**。

### 过程

1. **人类比较不同输出**：给同一个问题，让模型生成多个答案。人给这些答案排序（哪个更好）。
2. **训练奖励模型**：根据人的排序，训练一个"奖励模型"——它能预测人对输出的评分。
3. **用奖励模型优化主模型**：主模型每次生成回答后，奖励模型给它打分。主模型的目标是获得更高的分数。

```
给模型一个输入
    ↓
模型生成回答 A、B、C
    ↓
人排序：B > A > C
    ↓
训练"奖励模型"学习人的偏好
    ↓
用奖励模型的分数来优化主模型
```

这个过程让模型的输出风格更符合人类的期待——更简洁、更准确、更有帮助。

---

## 三个关键概念一览

| 概念 | 一句话 | 类比 |
|------|--------|------|
| **预训练** | 从海量文本中学到语言的统计规律 | 一个人读了几百万本书，学会了语言的模式 |
| **SFT** | 用问答对让模型学会"回答问题" | 这个人练习过很多次"问 → 答" |
| **RLHF** | 根据人类反馈调整输出风格 | 每次回答后，别人告诉它"这样说更好" |
| **推理** | 用训练好的模型处理新的输入 | 你现在正在做的事——你问，模型答 |

## 回到基石篇

读到这里的你，应该已经能回答这些问题了：

| 基石篇的问题 | 现在你知道了 |
|-------------|-----------|
| LLM 是怎么"训练"出来的？ | 预训练 → SFT → RLHF 三步 |
| Token 是什么？ | 文本的基本单位，被映射成嵌入向量 |
| 为什么同样的 Token 在不同上下文中含义不同？ | 注意力机制让每个词的表示依赖于上下文 |
| 为什么 LLM 会"忘记"前面的话？ | 上下文窗口的限制——注意力机制只能看到窗口内的词 |
| 提示词工程为什么有效？ | 你给的 prompt 提供了上下文，注意力机制会聚焦在这些词上 |

**要点总结**
- LLM 的训练分三步：**预训练**（学语言）→ **SFT**（学对话）→ **RLHF**（学偏好）
- **预训练**做的事极其简单：预测下一个词。但规模极大——几千亿 token、上千张 GPU
- **SFT** 用人工编写的问答对让模型学会回答问题
- **RLHF** 根据人的反馈调整输出风格——让模型的回答更符合人类期望
- 这三步完成后，你拿到的是 ChatGPT 这样的对话助手——然后你在聊天时，是**推理阶段**，不是训练
