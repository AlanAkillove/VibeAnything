# 02-构建高质量信息管道：AI 时代的信息筛选术

> 读完本文你将了解：为什么信息过载在 AI 时代反而更严重、如何筛选高质量的信息源、用 RSS/ArXiv/Newsletter 构建信息管道的方法、以及 AI 如何帮你对抗信息过载

---

## 你可能遇到过的问题

信息越来越多，但高质量的内容越来越难找到：

- **每天刷 Twitter/X、微信订阅号、知乎、Reddit**——看了很多，但值得记住的东西少得可怜
- **收藏了上百篇文章/论文**——收藏即吃灰，再也没有翻看过
- **AI 可以帮你总结一切**——但你发现 AI 生成的摘要看多了，反而失去深度阅读的能力
- **不知道该关注谁**——技术领域 KOL 太多，高质量的来源难以辨别
- **FOMO (Fear Of Missing Out)** 严重——怕错过重要的论文或技术趋势，不停地刷新

信息过载不是信息太多的问题，而是**信号太弱、噪音太强**的问题。你缺的不是信息获取能力，而是信息筛选能力。

---

## 常见的误解

| 你以为的 | 实际的 |
|---------|--------|
| 信息越多越好，知道的总比不知道的强 | 低质量信息占据认知带宽，让你漏掉真正重要的信号。**少即是多** |
| AI 能帮我筛选信息 | AI 可以摘要、可以排序，但**筛选标准需要你来定**——AI 不知道什么是"对你重要" |
| 刷 Twitter/X 就是在获取最新技术动态 | 刷信息流 ≈ 被动接收算法推给你的内容，**高质量信息需要主动获取** |
| 每天看 ArXiv 最新论文就能跟上前沿 | ArXiv 每天几百篇新论文，直接刷等于大海捞针。需要**先过滤再阅读** |
| 收藏/稍后读就是学到了 | 收藏 ≠ 学习。没有消化和整理的信息，只是从别人的脑子复制到了你的收藏夹 |

---

## 核心原则：Pull，不要 Push

信息获取有两种模式：

**Push 模式**（你正在用的）：算法推送给你——Twitter/X feed、微信订阅号、知乎推荐、抖音。优点是省力，缺点是算法优先推荐"让你停留时间最长的内容"，而不是"对你最有价值的内容"。

**Pull 模式**（你应该用的）：你主动设置过滤器，每天固定时间拉取精选内容。优点是质量可控，缺点是前期需要自己搭建管道。

在 AI 时代，Pull 模式比以往任何时候都容易实现。核心思路是：**用人和 AI 做两层过滤，让高质量信息主动流到你面前**。

---

## 三层信息管道搭建指南

### 第一层：核心信源（人肉筛选）

这一层决定你信息管道的质量天花板。花时间认真选，不要随便关注。

**ArXiv 论文追踪**
- 关注方向：找到你研究领域最近 3-5 年的会议论文（NeurIPS / ICML / CVPR / ACL 等），追踪这些作者的 ArXiv 主页
- ArXiv 邮件订阅：设置你感兴趣的 cs.AI / cs.LG / cs.CL 等分类的每日摘要（[https://arxiv.org](https://arxiv.org)）
- Semantic Scholar (semanticscholar.org)：比 ArXiv 更好用的论文发现和推荐工具，可以 Follow 特定作者和话题

**Twitter/X 高质量账户**
- 原则：Follow 做研究的人，不要 befriend 内容搬运工
- 建议类型：论文作者本人、顶级实验室官方账号、领域内公认的技术 leader
- 利用 Twitter List：把不同领域的人分到不同的 List，每周集中看一次，不要每天刷时间线

**Newsletter**
- 优点：经过作者筛选的浓缩信息，比刷信息流高效得多
- AI/ML 推荐：[The Batch (DeepLearning.AI)](https://www.deeplearning.ai/the-batch/)、[Import AI (Jack Clark)](https://importai.substack.com/)、[TLDR AI](https://tldr.tech/ai)
- 通用技术：[Hacker Newsletter](https://hackernewsletter.com/)、[Changelog Weekly](https://changelog.com/weekly)

**RSS 是核心枢纽**
不要低估 RSS 的价值。在信息源分散的时代，RSS 是把你所有信源集中的唯一可靠方式。

```
推荐工具链：
Feedly / Inoreader → 免费 RSS 聚合器，支持分组、标签、搜索
RSSHub (https://rsshub.app) → 万物皆可 RSS，把 Twitter/X、B站、知乎、微信公众号都转成 RSS
Miniflux / FreshRSS → 自托管方案，适合对隐私和数据控制要求高的用户
```

### 第二层：AI 预过滤

信息管道最重要的是**减少你需要看的总量**，而不是提高你看的速度。AI 在第二层帮你大幅减负。

**ArXiv 论文预筛选**
```
用 AI 做每日论文过滤的 prompt：

"请从以下论文列表中，帮我筛选出与[你的研究方向]相关的论文。
对每篇候选论文给一个 1-5 分的*个人相关性评分*（不是论文质量评分）。
评分标准：
5 = 必读，和我当前的研究直接相关
4 = 值得精读，有潜在价值
3 = 略读摘要即可
2-1 = 跳过，相关性不高

对 4 分以上的论文，用一句话说明为什么它对我有价值。"
```

**长文/长视频 AI 预消化**
```
"请把这篇[文章/视频/论文]浓缩成 3 个核心观点。如果我觉得某个观点值得深挖，我会再问你细节。
浓缩时保留原文的论据和引用，不要只给我结论。"
```

**Twitter/X / Newsletter 的 AI 周报**
每周用 AI 整理你这一周收藏/保存的内容，按主题分类，标记出值得深读的：
```
"这是我这一周保存的[文章链接/笔记/收藏]，请帮我：
1. 按主题分类
2. 标记最值得深读的 3 篇
3. 对每篇标记一个关键词，方便我后续检索
4. 告诉我哪些内容之间可能有联系，可以形成一个知识主题"
```

### 第三层：消化与沉淀

信息经过前两层过滤后，还有最关键的一步：**消化**。

- **不要只看 AI 摘要**——摘要用于"决定是否要读"，不是"替代阅读"。需要精读的文章，请完整读完，然后用自己的话写笔记
- **用 Obsidian 做笔记**（参考本篇第四章"第二大脑与 PKM"的详细方法）
- **定期回顾**：每月花一次时间回顾你过去一个月的"过滤后信息"，看哪些值得长期保存，哪些只是过眼云烟
- **定期修剪信源**：每季度检查你的 RSS 订阅和关注的账户，取消那些你连续三个月都没认真读的信源

## 常见问题快速解决

| 问题 | 解决方案 |
|------|---------|
| ArXiv 每天几百篇论文，看不过来 | 用 Semantic Scholar 或 Arxiv Sanity Lite (karpathy 的工具) 做 initial filter，再结合 AI 预筛选 |
| 不想错过重要论文 | 设置论文邮件提醒（Google Scholar / ArXiv），加上 AI 周报兜底——好论文不会只出现一次 |
| 关注的 Twitter/X KOL 太多 | 建 Twitter List，分领域归档，每周看一次 List 而不是每天刷首页 |
| 收藏夹堆了几百条"稍后读" | 一次性导出到 AI，让 AI 帮你分类并标记优先级，然后**删除那些你不会真读的** |
| RSS 太麻烦，不想搭 | 从最简单的开始：注册 Feedly 免费版，订阅 2-3 个 Newsletters + 1 个 ArXiv digest，两周后看效果再决定要不要扩展 |
| AI 生成的摘要不可靠 | 摘要只做"决策要不要读"的参考，不做"替代阅读"用。关键论文/文章一定要读原文 |

---

## 具体做法：一周搭建你的信息管道

**Day 1**：注册 Feedly 免费版，订阅你研究方向最重要的 2 个 ArXiv 分类 + 1 个核心 Newsletter
**Day 2**：在 Twitter/X 上建一个 List，把你最信任的 10-15 位研究人员加进去
**Day 3**：写一条 AI prompt 用于每日论文预筛选，测试 3 天，调整到满意度
**Day 4**：建立"每周 AI 周报"的习惯：每周六花 15 分钟让 AI 整理这周保存的内容
**Day 5**：在 Obsidian 里建一个"阅读笔记"模板：标题、核心观点、我的思考、下一步行动
**Day 6-7**：常规运行，下周日回顾效果，调整信源

**要点总结**
- 信息过载不是信息太多，是**噪音太强、信号太弱**。解决方法是搭建"Pull 模式"的信息管道，用 RSS 聚合，主动拉取，而不是被动接收算法推送
- 三层管道：**第一层人肉筛选信源**（ArXiv / Twitter Lists / Newsletter / RSS），**第二层 AI 预过滤**（每日论文筛选 + AI 周报），**第三层消化沉淀**（精读 + 笔记 + 定期回顾）
- AI 只能帮你过滤和摘要，**筛选标准要你定**，**深度消化要你做**
- 从最简单的方案开始：Feedly + 3 个信源 + 1 条 AI prompt，两周后迭代优化
- 定期修剪信源：取消关注连续三个月都没读的内容——减少信源比增加阅读速度更有效
