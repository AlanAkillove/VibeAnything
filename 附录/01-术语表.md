# 01-术语表

> AI 领域术语中英对照表，包含常用中文译名、发音提示和简短解释。按字母顺序排列，方便快速查阅。

| 术语 | 中文译名 | 发音提示 | 简短解释 |
|------|---------|---------|---------|
| **Agent** | 智能体 | /ˈeɪdʒənt/（诶真特） | 能自主执行任务的 AI 系统，可调用工具、做决策、记忆上下文 |
| **Agent SDK** | 智能体开发套件 | /ˈeɪdʒənt es diː keɪ/（诶真特 S-D-K） | Anthropic 推出的 Agent 开发框架，支持 Claude Code 非交互式/CI/CD 自动化运行，独立 monthly credits 计费 |
| **API Key** | 接口密钥 | /ˌeɪ piː ˈaɪ kiː/（A-P-I key） | 调用 API 时的身份凭证，敏感信息，需保密管理 |
| **Attention** | 注意力机制 | /əˈtenʃn/（呃吞申） | 模型在处理序列时动态分配权重，决定不同位置的关注程度，Transformer 的核心 |
| **Base URL / Endpoint** | 接口地址 | /ˈbeɪs juːˈɑːrɛl/（贝斯 URL） | API 的访问地址，切换不同模型平台时主要修改的位置 |
| **Chain-of-Thought (CoT)** | 思维链 | /tʃeɪn əv θɔːt/（晨 饿夫 索特） | 引导模型逐步推理的 prompting 技术，分步输出思考过程 |
| **Chat Template** | 对话模板 | /tʃæt ˈtemplɪt/（柴特 坦普利特） | 将对话历史格式化为模型输入的结构化模板，不同模型格式各异 |
| **Context Window** | 上下文窗口 | /ˈkɑntekst ˈwɪndoʊ/（康泰克斯特 温抖） | 模型单次能处理的 token 上限，越长越能理解完整上下文 |
| **Deep Learning** | 深度学习 | /diːp ˈlɜːrnɪŋ/（迪普 乐宁） | 使用多层神经网络进行学习的 AI 方法，LLM 的技术基础 |
| **Embedding** | 向量嵌入 | /ɪmˈbedɪŋ/（因贝丁） | 将文字/图像转为数值向量，语义相近的内容向量距离也近 |
| **Few-shot** | 少样本 | /fjuː ʃɑt/（夫由 沙特） | 在 prompt 中提供少量示例来引导模型，提高任务表现 |
| **Fine-tuning** | 微调 | /ˈfaɪn tuːnɪŋ/（范 图宁） | 在预训练模型基础上用特定数据集继续训练，让模型适配特定任务 |
| **Function Calling** | 函数调用 | /ˈfʌŋkʃn ˈkɔːlɪŋ/（放克申 扣领） | 模型识别用户意图并输出结构化参数，由外部系统执行实际函数 |
| **Generation** | 生成 | /ˌdʒenəˈreɪʃn/（坚纳瑞深） | 模型逐 token 预测输出文本的过程，通常使用自回归方式 |
| **Hallucination** | 幻觉 | /həˌluːsɪˈneɪʃn/（哈鲁西内深） | 模型生成看似合理但事实错误的内容，无法彻底消除，只能缓解 |
| **Inference** | 推理 | /ˈɪnfərəns/（因佛润斯） | 模型接收输入并生成输出的过程，即「跑模型」的实际含义 |
| **LLM** | 大语言模型 | /ˌel el ˈem/（L-L-M） | Large Language Model，基于 Transformer 的文本生成模型 |
| **Logits** | 逻辑值 | /ˈlɒdʒɪts/（拉机次） | 模型输出层未归一化的原始分数，经 softmax 转为概率分布 |
| **Logprob** | 对数概率 | /ˈlɒɡprɑb/（拉各普拉布） | 模型给每个 token 的 log probability，反映模型对下一个 token 的置信度 |
| **LoRA** | 低秩适配 | /ˈlɔːrə/（劳拉） | Low-Rank Adaptation，高效微调方法，只训练少量参数就能适配新任务 |
| **MCP** | 模型上下文协议 | /em siː piː/（M-C-P） | Model Context Protocol，让 AI 直接读取文件、执行命令、调用外部工具的协议 |
| **Message** | 消息 | /ˈmesɪdʒ/（卖色只） | Chat API 中的单元，包含 role（system/user/assistant）和 content |
| **Model** | 模型 | /ˈmɑdl/（马斗） | 经过训练后具备文本生成能力的神经网络，GPT、Claude、DeepSeek 等都是具体模型 |
| **MoE** | 混合专家架构 | /moʊˈiː/（某伊） | Mixture of Experts，将模型拆分为多个专家子网络，每次推理只激活部分参数，2026年主流旗舰架构（DeepSeek V4、GLM-5.2、MiMo-V2.5、Qwen 3.5等均采用） |
| **Multimodal** | 多模态 | /ˌmʌltiˈmoʊdl/（猫提某斗） | 模型能同时处理文字、图像、音频、视频等多种输入形式 |
| **Neural Network** | 神经网络 | /ˈnʊrəl ˈnetwɜːrk/（努肉 内特沃克） | 受生物神经元启发的计算模型，大量参数通过训练调整连接权重 |
| **Output** | 输出 | /ˈaʊtpʊt/（奥普特） | 模型生成的文本内容，也称 completion 或 response |
| **Parameter** | 参数 | /pəˈræmɪtər/（颇软米特） | 模型内部的权重数值，参数越多模型越强，如 7B/70B 即 70 亿/700 亿参数 |
| **Pre-training** | 预训练 | /priː ˈtreɪnɪŋ/（普瑞 垂宁） | 在大规模语料上从零训练模型，学习语言的基础知识和模式 |
| **Prompt** | 提示词 | /prɑmpt/（普让普特） | 给模型的输入指令，质量直接影响输出质量，即「提示词工程」的 Prompt |
| **R1 / Reasoning Model** | 推理模型 | /ˈriːzənɪŋ/（瑞怎宁） | 在输出前进行内部推理/思考的模型，如 DeepSeek V4 Think模式、OpenAI o3。DeepSeek V4内置Non-Think/Think High/Think Max三级推理 |
| **RAG** | 检索增强生成 | /ræɡ/（瑞格） | Retrieval-Augmented Generation，从知识库检索相关内容后让模型基于检索结果回答 |
| **RLHF** | 基于人类反馈的强化学习 | /ɑːr el eɪtʃ ef/（R-L-H-F） | 用人类偏好数据微调模型，使其输出更符合人类期望，ChatGPT 成功的关键技术之一 |
| **Role** | 角色 | /roʊl/（肉） | Chat API 中消息的身份：system（系统指令）、user（用户）、assistant（模型） |
| **Sampling** | 采样 | /ˈsæmplɪŋ/（三普令） | 从模型预测的概率分布中选择下一个 token 的策略，如 top-k、top-p 采样 |
| **SFT** | 监督微调 | /es ef tiː/（S-F-T） | Supervised Fine-Tuning，用标注数据对模型进行有监督训练，对齐人类偏好 |
| **Streaming** | 流式输出 | /ˈstriːmɪŋ/（斯锤名） | 模型逐 token 实时返回输出，用户无需等待完整生成，体验更流畅 |
| **System Prompt** | 系统提示词 | /ˈsɪstəm prɑmpt/（西斯腾 普让普特） | 设定模型角色和行为的全局指令，优先于用户消息 |
| **Temperature** | 温度 | /ˈtemprətʃər/（谈普热车） | 控制输出随机性的参数：低温度更确定，高温度更多样 |
| **Think Mode** | 推理模式 | /θɪŋk moʊd/（辛克 某德） | 模型在输出最终答案前先进行内部推理（thinking/reasoning），DeepSeek V4、Claude Opus 等旗舰模型均支持，适合复杂逻辑任务 |
| **Token** | 词元 | /ˈtoʊkən/（透肯） | 模型处理文本的最小单位，不直接对应单词，1 token 大约 0.75 个英文单词或 1.5 个中文字 |
| **Tokenizer** | 分词器 | /ˈtoʊkənaɪzər/（透肯奈泽） | 将文本切分为 token 的工具，不同模型使用不同的 tokenizer |
| **Tool Use** | 工具调用 | /tuːl juːs/（图尔 由斯） | 模型主动选择调用外部工具（计算器、搜索、代码执行等）的能力 |
| **Top-K** | Top-K 采样 | /tɑp keɪ/（塔普 K） | 只从概率最高的 K 个 token 中采样，限制候选范围 |
| **Top-P** | Top-P 采样 | /tɑp piː/（塔普 P） | 从累计概率达到 P 的最小 token 集合中采样，也称 nucleus sampling |
| **Transformer** | Transformer 架构 | /trænsˈfɔːrmər/（穿丝佛么） | 2017 年提出的深度学习架构，以 Attention 为核心，LLM 的基础架构 |
| **Training** | 训练 | /ˈtreɪnɪŋ/（垂宁） | 让模型从数据中学习参数的过程，分为预训练、微调、RLHF 等阶段 |
| **Vision** | 视觉理解 | /ˈvɪʒn/（维真） | 模型理解和分析图像内容的能力，GPT-5.x、Claude Sonnet 4.6、Qwen-VL 等均支持 |
| **vLLM** | 高性能推理引擎 | /ˈviː ˌel el ˈem/（V-L-L-M） | 开源 LLM 推理框架，PagedAttention 技术大幅提升推理吞吐 |
| **Weight** | 权重 | /weɪt/（威特） | 神经网络中的可学习参数，模型的能力通过训练调整这些权重获得 |
| **Zero-shot** | 零样本 | /ˈzɪroʊ ʃɒt/（基柔 沙特） | 不提供任何示例，直接让模型理解任务要求并执行的能力 |

> 提示：不用一次性记住所有术语。遇到不认识的词再回来查，用得多自然就熟了。
