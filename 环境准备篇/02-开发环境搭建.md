# 02-开发环境搭建（VS Code、终端、Python）

> 读完本文你将了解：怎么搭建一个适合 AI 辅助开发的 Python 环境、VS Code 必备插件、终端基础操作、以及虚拟环境管理

---

## 你可能遇到过的问题

你跟着上一篇文章注册了 API Key，兴冲冲地想开始写代码：

- **"我应该用哪个编辑器？"**——有人推荐 VS Code，有人推荐 PyCharm，还有人用 Jupyter
- **"我把 openai 装上了，但运行报错 'No module named openai' "**——你明明装了，它却说找不到
- **"Python 版本有好多个，我该用哪个？"**——系统自带的 Python 2、官网装的 Python 3、Anaconda……搞不清
- **"别人说要用虚拟环境，但我不知道那是什么"**

这些都是开发环境的问题。花半小时搭好环境，省下来的时间是几倍。

---

## 常见的误解

| 你以为的 | 实际的 |
|---------|--------|
| 编辑器越强大越好 | 对初学者来说，**VS Code 是最好的选择**——功能够用、生态好、免费 |
| 在系统 Python 里直接装包就行 | **永远不要在系统 Python 里装包**——不同项目需要不同的包版本，用虚拟环境隔离 |
| Anaconda 很省事 | 对 AI 开发来说，Anaconda 太臃肿了。**venv（轻量）或 uv（快速）** 更适合 |

---

## 遇到这些问题怎么办？

| 你遇到的问题 | 现在可以怎么做 |
|-------------|---------------|
| 装好的包运行时报找不到 | 检查是不是激活了虚拟环境，或者是不是在正确的环境里安装的 |
| Python 版本太多搞不清 | 终端输入 `python --version` 看当前用的是哪个版本 |
| VS Code 装好后不知道怎么用 | 先装 Python 插件，然后打开一个 `.py` 文件，VS Code 会自动帮你配置好大部分东西 |

## 第一步：安装 VS Code

下载地址：[code.visualstudio.com](https://code.visualstudio.com)

安装完成后，打开 VS Code，在左侧插件栏（Extensions，快捷键 Ctrl+Shift+X）搜索并安装以下插件：

| 插件 | 作用 |
|------|------|
| **Python**（Microsoft 官方） | Python 语言支持（语法高亮、IntelliSense、调试），自动包含 Pylance |
| **Jupyter**（Microsoft 官方） | 在 VS Code 里运行 .ipynb 笔记本 |
| **Markdown All in One** | Markdown 编辑增强 |
| **GitLens** | Git 历史可视化（下一篇会细讲） |

注意：如果你在用 AI 编程工具（如 Codex CLI、Cursor），它们本身有独立的安装流程，VS Code 是你的"主编辑器"，AI 工具在你的项目目录下运行。

## 第二步：终端基础

终端（Terminal）是开发者最常用的工具之一。在 VS Code 里按 **`Ctrl+``**（反引号）可以打开内置终端。

你需要熟练使用以下命令：

### Windows（PowerShell）

| 操作 | 命令 |
|------|------|
| 查看当前目录 | `pwd` |
| 列出文件 | `ls` |
| 进入目录 | `cd 目录名` |
| 返回上级 | `cd ..` |
| 查看 Python 版本 | `python --version` |
| 查看当前环境 | `pip list` |

### macOS / Linux（bash）

| 操作 | 命令 |
|------|------|
| 查看当前目录 | `pwd` |
| 列出文件 | `ls -la` |
| 进入目录 | `cd 目录名` |
| 返回上级 | `cd ..` |
| 查看 Python 版本 | `python3 --version` |
| 查看当前环境 | `pip list` |

**不需要记住所有命令。** 用几次就熟了。不记得的时候可以问 AI：`"在 PowerShell 里怎么切换到 D 盘？"`

## 第三步：Python 环境搭建

### 3.1 确认 Python 已安装

打开终端，输入：

```bash
python --version
```

如果看到类似 `Python 3.10` 或 `3.11` 或 `3.12` 的输出，说明已经装好了。如果没装，去 [python.org](https://python.org) 下载安装最新版本（安装时勾选 "Add Python to PATH"）。

### 3.2 创建虚拟环境

**为什么需要虚拟环境？**

你会在不同项目里用到不同版本的包。项目 A 需要 `openai==1.0`，项目 B 需要 `openai==1.5`。如果你装在全局，就会冲突。

虚拟环境给每个项目一个独立的包空间。

### 方法 A：使用 venv（Python 自带，推荐新手）

```bash
# 在项目目录下执行
python -m venv .venv

# Windows（PowerShell）激活
.venv\Scripts\Activate.ps1

# 如果提示执行策略错误，先运行：Set-ExecutionPolicy RemoteSigned -Scope CurrentUser

# Windows（cmd.exe）激活
.venv\Scripts\activate

# macOS / Linux 激活
source .venv/bin/activate
```

激活后，终端前面会出现 `(.venv)` 的提示，说明已经在虚拟环境里了。

### 方法 B：使用 uv（更快，适合进阶）

`uv` 是一个 Rust 写的 Python 包管理器，比 pip 快很多。

```bash
# 安装 uv（推荐用独立安装器，uv 独立于 Python 环境）
powershell -c "irm https://astral.sh/uv/install.ps1 | iex"

# 或者用 pip 安装：
pip install uv

# 创建虚拟环境
uv venv

# 安装包（比 pip 快）
uv pip install openai requests python-dotenv

# 运行脚本
uv run python your_script.py
```

`uv venv` 会在当前目录创建 `.venv` 文件夹。激活方式和 venv 一样。

## 验证你的环境

创建一个 `test.py` 文件，写入：

```python
import requests
from openai import OpenAI
from dotenv import load_dotenv

print("所有库已就绪！")
```

然后在终端运行：

```bash
python test.py
```

如果输出 "所有库已就绪！"，说明环境配置成功。

常见问题：

| 报错 | 原因 | 解决 |
|------|------|------|
| `ModuleNotFoundError: No module named 'openai'` | 没装 `openai` 包 | `pip install openai` |
| `ModuleNotFoundError: No module named 'requests'` | 没装 `requests` 包 | `pip install requests` |
| `ModuleNotFoundError: No module named 'dotenv'` | 没装 `python-dotenv` 包 | `pip install python-dotenv` |
| 安装时报错 `pip: command not found` | Python 环境没配好 | 确认 Python 已安装且已加到 PATH |

1. 安装 VS Code 和 Python 插件

5. 运行上面的验证脚本确保成功
6. 把 VS Code 的终端打开，感受一下用终端操作的感觉

**要点总结**
- VS Code + Python 插件是最推荐的开发环境组合
- **用虚拟环境管理依赖**，不要在全局装包
- 推荐 `venv`（新手）或 `uv`（进阶），不建议用 Anaconda
- 终端不熟悉很正常，多用几次就熟了
- 遇到报错，先检查是否在正确的虚拟环境里
