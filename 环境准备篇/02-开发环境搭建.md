# 02-开发环境搭建（VS Code、终端、Python）

> 读完本文你将了解：如何搭建一套适合 AI 辅助开发的 Python 工作环境、VS Code 必备插件与 AI 功能配置、终端基础操作、以及虚拟环境的正确使用方式
> 注：本文面向零基础新手，全程采用国内可直接访问的工具与方案，无需特殊网络环境。

## 你可能遇到过的问题

你跟着上一篇文章注册好了 API Key，准备开始写代码调用大模型，却先卡在了环境上：

- **不知道选什么编辑器**——有人说 VS Code，有人推 PyCharm，还有人用 Jupyter，新手不知道从哪下手
- **明明装了库，运行却提示 No module named xxx**——搞不清装到了哪个 Python 环境里
- **电脑里 Python 版本乱七八糟**——系统自带的、微软商店装的、官网下载的，不知道该用哪个
- **总听人说虚拟环境，但不知道到底有什么用、怎么操作**
- **AI 编程插件怎么配？代码补全、项目级 AI 助手怎么用**

这些都是开发环境的基础问题。花半小时把环境搭顺，后续写代码能省下几倍的排查时间。

## 常见的误解

| 你以为的 | 实际的 |
|---------|--------|
| 编辑器功能越全越好 | 对 AI 开发初学者来说，**VS Code 是最优选择**——免费开源、生态完善、AI 工具支持最好，足够覆盖 99% 的开发场景 |
| 直接在系统 Python 里装包就行 | **永远不要在系统全局 Python 里装项目依赖**——不同项目需要不同版本的库，全局安装会出现版本冲突，甚至搞坏系统环境 |
| Anaconda 最省事，新手必装 | 对大模型应用开发来说，Anaconda 过于臃肿，很多功能用不上。**venv（Python 自带，零学习成本）或 uv（极速现代工具）** 更轻量好用 |
| 有 AI 写代码就不用管环境了 | AI 只能帮你写代码，运行和调试依然依赖本地环境，环境问题 AI 也没法隔空解决 |

## 常见问题快速排查

| 你遇到的问题 | 快速解决方向 |
|-------------|-------------|
| 装好的库运行时找不到 | 90% 是没激活对应虚拟环境，或者 VS Code 没选对解释器 |
| Python 版本太多搞不清 | 终端输入 `python --version` 查看当前生效版本；用 `uv python` 可统一管理多版本 |
| VS Code 写 Python 没有补全 | 先安装官方 Python 插件，再在右下角选择对应虚拟环境的解释器 |
| PowerShell 激活虚拟环境报错 | 执行一次 `Set-ExecutionPolicy RemoteSigned -Scope CurrentUser` 放开执行策略 |

## 第一步：安装 VS Code

下载地址：[code.visualstudio.com](https://code.visualstudio.com)，国内可直接访问，按系统下载安装即可。

安装完成后，打开 VS Code，在左侧插件栏（Extensions，快捷键 `Ctrl+Shift+X`）搜索安装以下核心插件：

| 插件 | 核心作用 |
|------|---------|
| **Python**（Microsoft 官方） | Python 语言核心支持，包含语法高亮、智能补全、调试、代码跳转，已内置 Pylance 引擎 |
| **Jupyter**（Microsoft 官方） | 在 VS Code 内运行 `.ipynb` 笔记本，适合数据分析、代码片段调试 |
| **GitHub Copilot** | 官方 AI 编程助手，支持行内补全、代码对话、Agent 模式，可切换 Claude/GPT 等模型 |
| **GitLens** | Git 提交历史可视化，后续版本管理章节会用到 |
| **Markdown All in One** | Markdown 编辑增强，写笔记、文档更高效 |

> 💡 补充：当前 VS Code 已原生支持 **Agent 模式**与 **Model Context Protocol（MCP）**，可以让 AI 直接读取项目代码、执行终端命令、根据编译报错自动修复，是 AI 辅助开发的核心能力。Copilot 需绑定 GitHub 账号，学生可申请免费教育权益。

## 第二步：终端基础

终端（Terminal）是开发者的核心工具，在 VS Code 内按 **`Ctrl+``**（键盘左上角数字 1 旁边的反引号）即可快速调出内置终端，无需切换窗口。

你不需要记住大量命令，先掌握以下最常用的几个即可：

### Windows（推荐 PowerShell）
| 操作 | 命令 |
|------|------|
| 查看当前所在目录 | `pwd` |
| 列出当前目录下的文件 | `ls` |
| 进入指定目录 | `cd 目录名` |
| 返回上一级目录 | `cd ..` |
| 查看当前 Python 版本 | `python --version` |
| 查看当前环境已安装的包 | `pip list` |

### macOS / Linux（bash/zsh）
| 操作 | 命令 |
|------|------|
| 查看当前所在目录 | `pwd` |
| 列出当前目录下的文件（含隐藏文件） | `ls -la` |
| 进入指定目录 | `cd 目录名` |
| 返回上一级目录 | `cd ..` |
| 查看当前 Python 版本 | `python3 --version` |
| 查看当前环境已安装的包 | `pip list` |

> 💡 小技巧：命令不用死记硬背，用多了自然就熟了。遇到不会的操作，直接问 AI 即可，比如「PowerShell 里怎么切换到 D 盘的项目文件夹」。

## 第三步：Python 环境搭建

### 3.1 安装 Python
#### 版本选择
截至 2026 年 6 月，Python 最新稳定版为 **3.14.6**，3.12/3.13 是目前生态兼容性最好的版本。新手推荐安装 3.12 或 3.13 稳定版，避免最新版出现第三方库兼容问题。

#### 安装方式
- **Windows**：去 [python.org](https://python.org) 下载对应安装包，**安装时务必勾选「Add Python to PATH」**，否则终端识别不到 Python。也可通过微软商店安装，但更推荐官网版本。
- **macOS**：推荐通过 Homebrew 安装：`brew install python@3.12`；不建议使用系统自带的 Python。
- **Linux**：系统一般自带 Python 3，只需确保安装了 `pip` 即可。

安装完成后，打开终端输入 `python --version`，能正常输出版本号即安装成功。

### 3.2 为什么必须用虚拟环境？
每个项目依赖的库和版本都不一样：比如项目 A 需要 `openai 1.0`，项目 B 需要 `openai 1.5`，如果都装在全局 Python 里就会版本冲突，导致项目跑不起来。

虚拟环境就是给每个项目一个独立的「包空间」，互相隔离、互不影响。删除项目时直接删掉虚拟环境文件夹即可，不会污染系统环境。

### 方案 A：venv（Python 自带，新手首选）
venv 是 Python 标准库自带的虚拟环境工具，无需额外安装，零学习成本，最适合新手入门。

操作步骤（在项目文件夹内打开终端执行）：
```bash
# 1. 创建虚拟环境（会在当前目录生成 .venv 文件夹）
python -m venv .venv

# 2. 激活虚拟环境
# Windows PowerShell
.venv\Scripts\Activate.ps1

# Windows cmd
.venv\Scripts\activate.bat

# macOS / Linux
source .venv/bin/activate
```

激活成功后，终端提示符最前面会出现 `(.venv)` 标识，说明当前所有 pip 操作都会作用在这个虚拟环境里。

> 📌 注意：
> - Windows PowerShell 首次激活可能报错「禁止运行脚本」，先执行一次：`Set-ExecutionPolicy RemoteSigned -Scope CurrentUser`，选 Y 确认即可，只需设置一次。
> - Python 3.13 及以上版本，创建 venv 时会自动生成 `.gitignore`，避免把虚拟环境误提交到代码仓库。

### 方案 B：uv（现代极速工具，进阶推荐）
`uv` 是用 Rust 编写的现代 Python 包与环境管理工具，速度比传统 pip 快 10~100 倍，还能统一管理 Python 版本、依赖、虚拟环境，是目前 AI 开发圈的主流工具。

#### 安装 uv
Windows PowerShell 执行：
```powershell
powershell -ExecutionPolicy ByPass -c "irm https://astral.sh/uv/install.ps1 | iex"
```

macOS / Linux 执行：
```bash
curl -LsSf https://astral.sh/uv/install.sh | sh
```

安装完成后重启终端，输入 `uv --version` 验证是否成功。

#### 常用操作
```bash
# 1. 在当前项目创建虚拟环境（自动匹配 Python 版本）
uv venv

# 2. 安装依赖（比 pip 快数十倍）
uv pip install openai python-dotenv requests

# 3. 无需手动激活，直接运行脚本
uv run python test.py

# 额外能力：管理多版本 Python
uv python install 3.12  # 安装指定版本 Python
uv python pin 3.12      # 锁定当前项目的 Python 版本
```

uv 创建的虚拟环境同样是 `.venv` 文件夹，激活方式和原生 venv 完全一致，也 100% 兼容所有 pip 命令。

## 关键一步：VS Code 关联虚拟环境
这是新手最容易漏掉的一步：哪怕终端里激活了虚拟环境，VS Code 里如果没选对解释器，运行代码依然会报找不到库。

操作方法：
1. 打开项目里的任意 `.py` 文件
2. 看 VS Code 右下角状态栏，会显示当前的 Python 版本
3. 点击版本号，在弹出的列表里选择 `./.venv/Scripts/python.exe`（Windows）或 `./.venv/bin/python`（macOS/Linux）
4. 选择后，VS Code 的终端、调试、代码补全都将自动使用这个虚拟环境

## 验证你的环境
在项目里新建一个 `test_env.py` 文件，写入：
```python
import requests
from openai import OpenAI
from dotenv import load_dotenv

print("✅ 所有依赖加载成功，环境配置完成！")
```

先安装依赖：
```bash
# venv 环境
pip install openai python-dotenv requests

# uv 环境
uv pip install openai python-dotenv requests
```

然后运行脚本：
```bash
python test_env.py
```

如果输出成功提示，说明环境完全配置正确。

### 常见报错排查
| 报错信息 | 常见原因 | 解决方法 |
|---------|---------|---------|
| `ModuleNotFoundError: No module named 'openai'` | 当前环境没安装对应库，或没选对虚拟环境 | 先确认终端前有 `(.venv)` 标识，再执行安装命令；检查 VS Code 右下角解释器是否选对 |
| `ModuleNotFoundError: No module named 'dotenv'` | 包名是 `python-dotenv`，不是 `dotenv` | 执行 `pip install python-dotenv` |
| `pip : 无法将“pip”识别为 cmdlet` | Python 没加到 PATH，或没激活虚拟环境 | 检查 Python 安装时是否勾选了 Add to PATH，或先激活虚拟环境 |
| 激活脚本报错「无法加载，因为在此系统上禁止运行脚本」 | PowerShell 默认执行策略限制 | 执行 `Set-ExecutionPolicy RemoteSigned -Scope CurrentUser` 确认即可 |

## 你现在应该做什么？
1. 安装 VS Code，并装好 Python、Jupyter 等核心插件
2. 新建一个文件夹作为你的 AI 开发练习项目
3. 在项目内用 venv 或 uv 创建虚拟环境并激活
4. 安装 `openai`、`python-dotenv`、`requests` 三个基础库
5. 在 VS Code 右下角选择对应虚拟环境的解释器
6. 运行验证脚本，确保环境跑通
