## 你可能遇到过的问题

你用 AI 整理资料、写笔记、刷题，但很快遇到这些困扰：

- **AI 输出里全是 `#`、`*`、`` ` `` 符号**——看着乱，不知道是什么格式，复制到 Word 里排版全崩
- **笔记散落在 AI 对话框、便签、微信、Word 里**——想找某个知识点的时候，翻半天都找不到
- **总听人推荐 Obsidian**——但不知道它和印象笔记、Notion、OneNote 有什么本质区别
- **想把 AI 对话里的干货沉淀下来**——但不知道存成什么格式、放哪里，存完了也再也不会看
- **折腾了复杂的分类体系**——文件夹、标签、模板搞了一堆，最后还是记不住、用不起来

这些问题的最优解，就是 **Markdown 格式 + Obsidian 工具** 的组合。

## Markdown：AI 时代的通用文本语言

Markdown 是一种轻量级标记语言，设计目标就是「写起来像纯文本，读起来像排版文档」。

它是目前所有大模型的默认输出格式——无论是 GPT、Claude、DeepSeek 还是 Kimi，生成结构化内容时默认都会用 Markdown。同时 GitHub 文档、技术博客、笔记系统也全部支持 Markdown，相当于学会一次，所有场景通用。

### 核心语法（10 分钟掌握）
不用全背，优先记高频常用的，用几次就熟了：

| 显示效果 | Markdown 写法 | 使用频率 |
|---------|--------------|----------|
| 一级大标题 | `# 标题` | 极高 |
| 二级子标题 | `## 标题` | 极高 |
| 无序列表 | `- 列表项` | 极高 |
| 有序列表 | `1. 步骤一` | 高 |
| **加粗强调** | `**重点内容**` | 极高 |
| *斜体补充* | `*补充说明*` | 中 |
| 引用块 | `> 引用原文` | 高 |
| 行内代码 | `` `代码片段` `` | 高 |
| 带语法高亮的代码块 | ```python<br>print("hello")<br>``` | 极高（AI 写代码必用） |
| 超链接 | `[显示文字](链接地址)` | 中 |
| 表格 | `\| 列1 \| 列2 \| \n \|---\|---\|` | 中 |

学会这些，就能看懂 90% 的 AI 输出，也能写出清晰规整的技术笔记。

## Obsidian：属于你自己的本地知识库

Obsidian 是一款**基于纯 Markdown 文件的本地笔记应用**，核心理念是「你的数据永远属于你自己」。截至 2026 年，它已经成为技术学习者、研究者、开发者的主流笔记工具。

### 核心特性
| 特性 | 说明 |
|------|------|
| **本地优先** | 所有笔记存在你的本地硬盘，离线可用，没有会员也能访问全部内容，不会因为平台停服丢失数据 |
| **纯文件格式** | 每篇笔记就是一个独立的 `.md` 文件，可以用 VS Code、记事本等任何编辑器打开，也能直接用 Git 做版本管理 |
| **双向链接** | 用 `[[笔记标题]]` 就能快速链接到另一篇笔记，自动建立知识关联，比硬建文件夹分类灵活得多 |
| **全局搜索** | 毫秒级全文检索所有笔记，找关键词比翻聊天记录、翻文件夹快 10 倍 |
| **Canvas 画布** | 无限白板功能，用来梳理论文框架、画知识脉络、整理项目思路非常方便 |
| **插件生态** | 上千款社区插件按需扩展，比如任务管理、数据库、日历笔记，不用可以完全不装 |
| **可选同步服务** | 官方 Obsidian Sync 支持端到端加密多设备同步，自带一年版本历史；也可以自己用 Git、云盘同步，方案完全自由 |
| **内容发布** | Obsidian Publish 可一键把笔记发布成公开知识库、个人博客，不用额外搭网站 |

### 为什么计算机/AI 学习者特别适合用 Obsidian
1. **格式完全统一**：和 AI 输出、GitHub 文档、代码注释的格式天然一致，不用来回转换
2. **融入开发工作流**：笔记就是纯文件，可以直接用 VS Code 编辑，用 Git 做版本管理，推到 GitHub 备份，和你写代码的流程完全打通
3. **沉淀知识体系**：算法、专业课、论文、项目经验都可以沉淀成互联的笔记，复习时能顺着关联脉络完整梳理
4. **数据长期可控**：不会因为工具停服、会员过期丢失笔记，大学四年的积累可以一直带走

### 安装与入门步骤
1. 去官网 [obsidian.md](https://obsidian.md) 下载安装，全平台支持，个人非商业使用完全免费
2. 打开后创建一个 Vault（仓库），本质就是选一个本地文件夹存放你的所有笔记
3. 新建第一篇笔记，写几句 Markdown 试试渲染效果
4. 记住两个核心快捷键：`Ctrl+O` 快速搜索打开笔记、`Ctrl+Shift+F` 全局全文搜索
5. 用 `[[另一篇笔记名]]` 创建一条双链，打开左侧 Graph View 看看关联效果

> 💡 新手避坑提醒：不要一开始就装几十款插件、设计复杂的分类体系。先把内容记起来，缺什么功能再加什么，知识体系是慢慢长出来的，不是一开始设计出来的。

## AI + Obsidian 实用工作流

### 工作流 1：论文/技术文档精读
1. 把论文/长文档发给 AI，让它按 Markdown 格式输出核心论点、关键概念、疑问点、延伸方向
2. 在 Obsidian 新建笔记，标题用论文全名，粘贴 AI 整理的内容
3. 加上标签分类，比如 `#论文笔记 #大模型`
4. 把文中的关键概念用 `[[概念名]]` 链接到对应的基础知识笔记
5. 后续复习时，顺着双链就能把相关知识点串联成完整体系

### 工作流 2：算法刷题沉淀
1. 做完一道算法题，把题目、思路、代码模板发给 AI 整理成规范格式
2. AI 返回题解后，复制到 Obsidian，按「题目名-题型」命名
3. 加上标签，比如 `#算法 #动态规划`
4. 遇到同类型题目时用双链互相引用，慢慢形成自己的题型方法库
5. 整个笔记库可以直接用 Git 提交备份，换设备也不会丢失

### 工作流 3：课程笔记整理
1. 上课时先快速记录要点草稿，不用追求排版
2. 课后把草稿丢给 AI，让它补充细节、梳理结构、生成标准 Markdown
3. 整理后的内容粘贴回 Obsidian，补充上自己的理解和疑问
4. 用标签和双链关联前置知识，逐步形成完整的课程知识树
