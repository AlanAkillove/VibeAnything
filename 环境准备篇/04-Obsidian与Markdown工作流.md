# 04-Obsidian 与 Markdown 工作流

> 读完本文你将了解：Markdown 是什么、为什么它在 AI 时代这么重要、Obsidian 怎么帮你管理知识、以及怎么用 AI 辅助笔记本

---

## 你可能遇到过的问题

你在用 AI 做笔记、写文档、整理资料，但发现：

- **AI 给你的回答里有很多 `#`、`*`、`[]` 符号**——你不知道这些是干什么的，觉得"好乱"
- **你在网上看到很多人用 Obsidian 做笔记**，但不知道它和 OneNote / 印象笔记有什么区别
- **你的笔记越积越多，但想找什么东西的时候根本找不到**——分散在几十个文件里
- **你想把和 AI 的对话中有价值的部分保存下来**，但不知道用什么格式、存在哪里

这些问题都可以用一个组合解决：**Markdown + Obsidian**。

---

## 常见的误解

| 你以为的 | 实际的 |
|---------|--------|
| Markdown 是一种编程语言 | Markdown 是**轻量级标记语言**——它的设计目标就是"写起来像纯文本，看起来像格式化文档" |
| Obsidian 和 OneNote 差不多 | Obsidian 是**本地优先、基于文件的笔记工具**——你的笔记就是 `.md` 文件，不依赖任何云端格式 |
| 做笔记应该用专门的 AI 笔记工具 | Obsidian 本身不内置 AI，但你可以把 AI 的输出复制进来，或者用第三方插件接入 AI |
| Markdown 很难学 | Markdown 的核心语法 **10 分钟就能学会**——只有几个符号要记住 |

---

## 遇到这些问题怎么办？

| 你遇到的问题 | 现在可以怎么做 |
|-------------|---------------|
| AI 输出的格式看不懂 | 那些 `#`、`*`、`[]` 就是 Markdown——把它复制到 Obsidian 里，自动显示为格式化文本 |
| 笔记太多找不到 | Obsidian 的搜索（Ctrl+Shift+F）可以搜所有笔记；双链可以建立关联 |
| 想保存 AI 对话中有价值的部分 | 直接把 AI 的回答复制到一个 `.md` 文件，标题写清楚，想找的时候搜关键词 |
| 想做知识体系但不知道从哪开始 | 先不要追求"体系"，随便记。Obsidian 的画布和关系图谱会自动帮你发现关联 |

## Markdown：AI 时代的通用语言

Markdown 是 AI 最常用的输出格式。当 AI "写"东西的时候，它**几乎总是输出 Markdown**。

### 核心语法（10 分钟学会）

| 你要的效果 | 输入 | 输出效果 |
|-----------|------|---------|
| 大标题 | `# 标题` | # 标题 |
| 子标题 | `## 标题` | ## 标题 |
| 列表 | `- 项目` | - 项目 |
| 数字列表 | `1. 第一步` | 1. 第一步 |
| **加粗** | `**文字**` | **文字** |
| *斜体* | `*文字*` | *文字* |
| 链接 | `[显示文字](https://url.com)` | [显示文字](url.com) |
| 图片 | `![描述](图片链接)` | 显示图片 |
| 代码块 | \`\`\`python \n print("hi") \n \`\`\` | 带语法高亮的代码块 |
| 表格 | `\| A \| B \| \n \|--\|--\|` | 生成表格 |
| 引用 | `> 引用文字` | 灰色引用块 |

### 为什么在 AI 时代重要

无论是 ChatGPT、Claude、DeepSeek 还是 Kimi，AI 的输出格式都是 Markdown。这意味着你不需要学会一种新的"AI 输出格式"——学会 Markdown 就够了。

同时，GitHub 的所有文档（README、Wiki、Issue）都使用 Markdown。学会 Markdown = 学会和 AI 交流 + 学会和开发者社区交流。

## Obsidian：你的本地知识库

Obsidian 是一个**基于 Markdown 文件的本地笔记应用**。它的核心理念：

| 特性 | 说明 |
|------|------|
| **本地优先** | 笔记存在你的电脑上，不是云端 |
| **纯文件** | 每个笔记就是一个 `.md` 文件 |
| **双链** | 用 `[[笔记名]]` 连接不同笔记，自动建立关联图 |
| **搜索** | 全文搜索所有笔记，支持正则表达式 |
| **插件** | 丰富的社区插件生态 |

### 为什么 CS 学生应该用 Obsidian

- **你的笔记就是文件**——可以用 VS Code 打开、可以用 Git 版本管理、可以批量处理
- **AI 友好**——直接复制 AI 的输出，不需要格式转换
- **适合做知识积累**——大学四年你会读很多论文、写很多代码、积累很多笔记，Obsidian 让这些知识互联起来

### 安装和基本设置

1. 从 [obsidian.md](https://obsidian.md) 下载安装（Windows / macOS / Linux 均可）
2. 创建一个新的 Vault（仓库），选一个本地文件夹
3. 新建一个笔记，试试写点 Markdown
4. 按 `Ctrl+O` 快速切换笔记
5. 按 `Ctrl+Shift+F` 搜索所有笔记内容

### 几个值得关注的功能

| 功能 | 用途 |
|------|------|
| **双链 `[[]]`** | 在笔记里输入 `[[另一篇笔记]]`，自动创建链接 |
| **关系图谱** | 点击左侧 Graph View，看到所有笔记的关联网络 |
| **标签 `#tag`** | 用标签分类，比如 `#AI`、`#论文笔记` |
| **Callout** | `> [!NOTE]` 生成醒目的提示框（Obsidian 原生支持） |
| **Canvas** | 白板模式，用来画思维导图、流程图 |

## 一个实际工作流

### 场景：用 AI 帮你理解一篇论文

```
1. 把论文 PDF 发给 AI，让它帮你总结要点
2. AI 用 Markdown 格式返回总结
3. 复制到 Obsidian，文件名为 "论文标题.md"
4. 用标签 `#论文笔记` 标记
5. 在笔记中用 `[[相关概念]]` 链接到你已经有的笔记
6. 下次想找这篇论文，直接搜标题或标签
```

### 场景：用 AI 辅助上课笔记

```
1. 上课时在 Obsidian 里快速记笔记（Markdown 格式）
2. 课后把笔记发给 AI，让它补充和整理
3. AI 返回整理后的版本
4. 复制回 Obsidian，替换原始笔记
5. 用 `#课程笔记` 标签标记
```

## 你现在应该做什么？

1. 安装 Obsidian，创建一个 Vault
2. 新建一个笔记，标题写 "Markdown 速查"
3. 把本文的 Markdown 语法表复制进去
4. 新建第二个笔记，写你今天学到的一个东西
5. 在第一个笔记里用 `[[]]` 链接到第二个笔记
6. 打开关系图谱（Graph View），看看两个笔记之间的连线
