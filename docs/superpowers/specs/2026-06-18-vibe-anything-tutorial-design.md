# VibeAnything Tutorial — Design Spec

> 一个面向 CS 本科生的 Vibe Coding 入门教程仓库
> 时间：2026-06-18

## 1. 背景与目标

### 1.1 为什么做这个项目
大一大二的 CS 本科生对 AI 的认知大多停留在网页版 ChatGPT。在 Vibe Coding 时代，AI 正在快速重塑软件开发范式，但学校课程对此几乎没有覆盖。本教程旨在作为一份**系统性的入门指南**，帮助后辈在正确的方向上起步，避免自己摸索时踩不必要的坑。

### 1.2 目标读者画像
- 大一/大二 CS 本科生
- 有 Python 基础，无机器学习背景
- 对 LLM 了解仅限于网页版聊天
- 有一定的自学意愿，但缺乏信息筛选能力
- 面临未来的科研、求职、项目开发需求

### 1.3 核心理念
- **Vibe Coding 不是魔法，而是新技能**——AI 不会取代程序员，但会用 AI 的程序员会取代不会用的
- **基础知识不过时**——算法、系统设计、安全、测试在 AI 时代反而更重要
- **做中学**——不要从理论开始，从做一个东西开始
- **不回避现实**——包括网络问题、成本问题、AI 的局限和坑

## 2. 交付形式

- **GitHub 仓库**，以 Markdown 文档组织
- 语言：中英夹杂（术语保留英文，解释用中文）
- 结构：**模块化**，各模块独立，互不依赖
- 额外文件：`入门导航.md` 作为决策树，引导读者根据目标选择阅读路径

## 4. 设计原则与写作规范

### 4.1 写作风格
- 每篇文章控制在 1,500~3,000 字
- 中英夹杂：术语保留英文（LLM、Token、Agent、Prompt），解释用中文
- 多用实际案例和类比，少用抽象理论
- 每篇文章开头有"读完本文你将了解"的目标陈述
- 每篇文章末尾有"要点总结"或"思考题"
- 代码示例以 fenced code block 呈现，标注语言

### 4.2 难度控制
- 基石篇 / 环境准备篇：零基础可读
- 工具篇 / AI协作心法篇 / 工作流篇：有一定基础
- 进阶篇 / 科研篇：需要前面基础

### 4.3 更新策略
- 模块间尽量无交叉引用，避免断链
- 术语表统一维护，各篇术语链接回附录术语表
- 工具 / 模型的信息保持时效性标注

## 5. 未来可能的扩展方向
- 配套 Slide 用于线下分享
- 配套视频讲解
- 交互式练习（Jupyter Notebook）
- 更多的完整项目案例

## 6. 不包含的内容（范围边界）
- 不深入 ML / DL 理论（预备知识有独立章节）
- 不教具体的编程语言（假设读者已有 Python 基础）
- 不涉及模型训练、微调（进阶篇可选）
- 不涉及非 CS 领域的 AI 应用
