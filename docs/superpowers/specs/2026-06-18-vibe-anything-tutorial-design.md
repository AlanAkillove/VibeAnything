---

## 1. 背景与目标

### 1.1 为什么做这个项目
大一大二的 CS 本科生对 AI 的认知大多停留在网页版 ChatGPT。在 Vibe Coding 时代，AI 正在快速重塑软件开发范式，但学校课程对此几乎没有覆盖。本项目旨在作为一份**系统性的入门指南**，帮助后辈在正确的方向上起步。

### 1.2 目标读者画像
- 大一/大二 CS 本科生
- 有 Python 基础，无机器学习背景
- 对 LLM 了解仅限于网页版聊天
- 面临未来的科研、求职、项目开发需求
- 预算有限（¥0-100/月），无国际支付手段
- 网络受限，部分国际 AI 工具无法直接访问

### 1.3 核心理念
- **Vibe Coding 是新技能，不是魔法**——AI 不会取代程序员，但会用 AI 的程序员会取代不会用的
- **基础知识不过时**——算法、系统设计、安全、测试在 AI 时代反而更重要
- **做中学**——不要从理论开始，从做一个东西开始
- **不回避现实**——包括网络问题、成本问题、AI 的局限和坑
- **不 gatekeep**——如果资源有中文版就链接中文版，没有就提供中文总结

## 2. 交付形式

- **GitHub 仓库**，以 Markdown 文档组织
- 语言：中英夹杂（术语保留英文如 LLM/Token/Agent/Prompt，解释用中文）
- 结构：**模块化**，各模块独立，互不依赖
- 特色文件：`入门导航.md` 作为决策树，引导读者根据目标选择阅读路径
- 每篇文章 1,500-3,000 字，开头有目标陈述，结尾有要点总结

## 5. 写作规范

- 每篇文章 1,500-3,000 字
- 中英夹杂：术语保留英文，解释用中文
- 不出现参数更新公式，不手写反向传播
- 核心概念配生活类比→正式说法
- 每篇文章末尾链接"想深入了解"的外部资源
- 中国语境下的代码示例优先使用国内平台 API（如 SiliconFlow），同时展示国际 API 的等效写法
